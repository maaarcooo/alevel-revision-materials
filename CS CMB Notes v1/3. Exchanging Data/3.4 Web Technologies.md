# 1.3.4 Web Technologies - Revision Notes

## HTML (HyperText Markup Language)

**Purpose**: Language for structuring web pages using tags in angle brackets `<tag>`.

### Structure
- **`<html>`** - Wraps all HTML content
- **`<head>`** - Contains metadata (title, links to CSS)
- **`<body>`** - Contains visible page content

### Key HTML Tags

| Tag | Purpose |
|-----|---------|
| `<title>` | Text displayed in browser tab |
| `<h1>`, `<h2>`, `<h3>` | Headings in decreasing size |
| `<p>` | Paragraph with line spacing |
| `<img>` | Self-closing image tag (requires `src`, `height`, `width`) |
| `<a href="url">` | Hyperlink (anchor tag) |
| `<ol>` | Ordered (numbered) list |
| `<ul>` | Unordered (bulleted) list |
| `<li>` | List item |
| `<div>` | Division - separates page into distinct areas |
| `<link>` | Links external files (e.g., CSS stylesheets) |

### Classes and Identifiers

**Classes** (defined with `.classname`)
- Multiple elements can share the same class
- Used for consistent styling across elements

**Identifiers** (defined with `#idname`)
- Unique to one element
- Only one element can have a particular identifier

## CSS (Cascading Style Sheets)

**Purpose**: Language for describing webpage styling and appearance.

### Implementation Methods

**Internal/Embedded CSS**
- Placed within `<style>` tags in the HTML document
- Directly in the HTML file

**External CSS**
- Written in separate `.css` file
- Linked in HTML head: `<link href="stylesheet.css" rel="stylesheet" type="text/css">`

### CSS Syntax
```css
selector {
    property: value;
    property: value;
}
```

**Example**:
```css
body {
    margin: 0px;
    background-color: white;
    font-family: Arial;
    font-size: 18px;
    text-align: center;
}
```

## JavaScript

**Purpose**: Scripting language that adds **interactivity** to websites.

### Key Features
- **Interpreted** (not compiled) - runs in browser each time page loads
- **Platform-independent** - works regardless of processor architecture
- Commonly used for **client-side input validation**

### Advantages
- Local computer validates data before sending to server
- Reduces load on busy servers
- Reduces web traffic
- Immediate response to user actions
- Quick execution
- Greater developer control over website behaviour

### Common JavaScript Functions

**Changing HTML element attributes**:
```javascript
chosenElement = document.getElementById("example");
chosenElement.innerHTML = "Hello World";
```

**Writing to document**:
```javascript
document.write("Hello World");
```

**Alert box**:
```javascript
alert("Hello World");
```

## Search Engine Indexing

### Search Engines
**Definition**: Program that searches through a database of internet addresses for resources based on user criteria.

### Web Crawlers (Spiders)
- Collect information about websites to build search index
- Traverse the internet page-by-page following links
- Collect **keywords and phrases** from pages
- Collect **metadata** (information specified by website owner)
- Add all information to the index

### Importance
- Display order determines which websites users visit
- High-quality, relevant websites should appear first

---

## PageRank Algorithm

**Purpose**: Ranks web pages to determine search result display order. Higher rank = displayed first.

### Ranking Factors
1. **Number of incoming links** from other pages
2. **Page rank of pages** that link to it

### Data Structure
**Directed graph**:
- Nodes = web pages
- Arcs = links between pages

### PageRank Formula
```
PageRank(x) = (1-d) + d[(PageRank(T1) รท Count(T1)) + ... + (PageRank(Tn) รท Count(Tn))]
```

Where:
- **PageRank(x)** = page rank of page x
- **Count(Tn)** = total outbound links from page n
- **d** = damping factor (usually **0.85**)

### Calculation Process
1. **First iteration**: Assume page rank of 1 for unknown values
2. **Subsequent iterations**: Use calculated values from previous iteration
3. **Repeat** until values converge

**Note**: Modern search engines use more complex algorithms considering factors like page age, load speed, and content quality.

## Server-Side vs Client-Side Processing

### Server-Side Processing

**Definition**: Client sends data to server for processing. No processing on client computer.

**Languages**: SQL, PHP

**Advantages**:
- No plugins required
- Faster for large calculations (server more powerful)
- Not browser-dependent
- **More secure**

**Use case**: When security is critical or heavy computation needed

---

### Client-Side Processing

**Definition**: Client processes data on local device. All processing on client computer.

**Languages**: JavaScript

**Advantages**:
- Immediate response to user actions
- Quick execution
- More developer control over website behaviour and appearance
- Reduces server load

**Use case**: Input validation, interactive elements, immediate user feedback

---

## Summary Table: Key Comparisons

| Aspect | Server-Side | Client-Side |
|--------|-------------|-------------|
| **Processing location** | Server | Local device |
| **Languages** | SQL, PHP | JavaScript |
| **Speed** | Faster for large tasks | Faster for small tasks |
| **Security** | More secure | Less secure |
| **Browser dependency** | Independent | May vary |
| **Server load** | Higher | Lower |
| **User experience** | May have delays | Immediate response |

---

## Key Exam Points

1. **HTML** structures content, **CSS** styles it, **JavaScript** makes it interactive
2. Classes use `.` and can be reused; Identifiers use `#` and are unique
3. External CSS is better for maintaining consistent styling across multiple pages
4. JavaScript is **interpreted**, not compiled
5. Web crawlers follow links to build search engine indices
6. PageRank considers both **quantity** and **quality** of incoming links
7. PageRank uses **iterative calculation** with damping factor (typically 0.85)
8. Server-side is more **secure**; client-side provides **faster user response**
9. Modern search algorithms are much more complex than basic PageRank
