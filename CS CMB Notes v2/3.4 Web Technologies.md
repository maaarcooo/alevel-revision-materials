## HTML (HyperText Markup Language)

**HTML** is a **markup language** used to describe the **structure and content** of web pages. Browsers **interpret and render** HTML to display web pages. HTML uses **tags** written in **angle brackets** (`<tag>`, `</tag>`).

### Document Structure

A web page has two main sections:
- **`<head>`** - Contains metadata and the page title (displayed in browser tab)
- **`<body>`** - Contains the visible content of the webpage

### Essential HTML Tags

| Tag | Purpose |
|-----|---------|
| `<html>` | Root element - all HTML code goes within these tags |
| `<head>` | Defines the browser tab/window heading area |
| `<title>` | Defines text appearing in the browser tab |
| `<body>` | Defines the main content area |
| `<h1>` to `<h6>` | Heading styles in decreasing sizes |
| `<p>` | Defines a paragraph with spacing above and below |
| `<a>` | Anchor tag - defines a hyperlink |
| `<img>` | Embeds an image (self-closing tag) |
| `<ol>` | Ordered (numbered) list |
| `<ul>` | Unordered (bulleted) list |
| `<li>` | List item (used within `<ol>` or `<ul>`) |
| `<div>` | Divides page into separate named areas |
| `<link>` | Links to external files (e.g., CSS stylesheets) |

### Tag Examples

**Hyperlink syntax:**
```html
<a href="https://example.com">Link Text</a>
```

**Image syntax (self-closing - no closing tag required):**
```html
<img src="image.png" width="500" height="300">
```
Required attributes: `src` (source path), `width`, `height`

**Division with ID:**
```html
<div id="section1">Content here</div>
```

### Classes and Identifiers

Classes and identifiers are **attributes** assigned to HTML elements for styling purposes.

**Classes** (defined with `.` in CSS):
- Can be applied to **multiple elements** across web pages
- Enables **consistent styling** across elements
- Styling only needs to be defined **once**

```css
.highlight {
    background-color: yellow;
    font-weight: bold;
}
```
```html
<p class="highlight">This paragraph uses the class</p>
<p class="highlight">Multiple elements can share the same class</p>
```

**Identifiers/IDs** (defined with `#` in CSS):
- **Unique** - only **one element** can have a particular ID per page
- Used for styling or targeting specific elements

```css
#header {
    background-color: blue;
    color: white;
}
```
```html
<div id="header">Only one element can have this ID</div>
```

## CSS (Cascading Style Sheets)

**CSS** is a **language** used to describe the **style and appearance** of web pages. CSS specifies how HTML elements look and can be applied to tags, classes, and IDs.

### CSS Implementation Methods

| Method | Description |
|--------|-------------|
| **Internal/Embedded CSS** | Written inside `<style>` tags within the HTML document |
| **External CSS** | Written in a separate `.css` file and linked to HTML |

**Linking external stylesheet:**
```html
<link href="styles.css" rel="stylesheet" type="text/css">
```

### CSS Syntax

```css
selector {
    property: value;
    property: value;
}
```

**Example:**
```css
body {
    margin: 0px;
    padding: 0px;
    background-color: white;
    font-family: Arial, Helvetica, sans-serif;
    font-size: 18px;
    text-align: center;
}
```

Each CSS rule block:
1. Begins with a **selector** (element name, class, or ID)
2. Followed by **curly brackets** `{ }`
3. Contains **property-value pairs** separated by semicolons

## JavaScript

**JavaScript** is a **scripting language** used to add **interactivity** to websites. It is an **interpreted language** (not compiled), meaning browsers interpret the code each time a page loads, **independently of processor architecture**.

### Key Characteristics
- **Multi-paradigm language** (supports procedural, object-oriented, functional styles)
- Commonly used for **client-side input validation**
- Executed by the browser

### Advantages of JavaScript
- Local computer handles invalid data **before sending to server**
- **Reduces load** on busy servers
- **Reduces network traffic**

### JavaScript Output Methods

**Modifying HTML element content:**
```javascript
chosenElement = document.getElementById("example");
chosenElement.innerHTML = "Hello World";
```

**Writing directly to the document:**
```javascript
document.write("Hello World");
```

**Displaying an alert box:**
```javascript
alert("Hello World");
```

## Search Engine Indexing

### Search Engines

A **search engine** is a program that **searches through a database of internet addresses** looking for resources based on criteria set by the client.

The **order of displayed results** is crucial as it determines which websites users visit. Search engines aim to display **high-quality, relevant content** at the top.

### Web Crawlers (Spiders)

**Web crawlers** (also called **spiders**) collect information about websites to build the search engine's **index**.

**How web crawlers work:**
1. **Traverse** the internet page by page using **hyperlinks**
2. Collect **keywords and phrases** from web pages
3. Add this information to the search engine's **index**
4. Collect and store **metadata** (information specified by website owners)

## PageRank Algorithm

The **PageRank algorithm** ranks web pages to determine the **order of search results**. Higher-ranked pages appear first.

### Factors Determining PageRank

1. **Number of incoming links** from other web pages
2. **PageRank of the linking pages** - links from high-ranked pages carry more weight

### Data Structure

PageRank uses a **directed graph** where:
- **Nodes** represent web pages
- **Arcs** (directed edges) represent links between pages

### The PageRank Formula

$$PR(x) = (1-d) + d \times \left[\frac{PR(T_1)}{C(T_1)} + \frac{PR(T_2)}{C(T_2)} + ... + \frac{PR(T_n)}{C(T_n)}\right]$$

Where:
- **PR(x)** = PageRank of page x
- **d** = Damping factor (typically **0.85**)
- **T₁, T₂, ... Tₙ** = Pages that link to page x
- **C(Tₙ)** = Total count of **outbound links** from page Tₙ

### Calculation Process

1. **First iteration**: Assume initial PageRank of **1** for all pages
2. Calculate new PageRank values using the formula
3. **Subsequent iterations**: Use calculated values from previous iteration
4. Repeat until values **converge** (stabilise)

### Worked Example

For a graph with pages A, B, C, D, E where:
- A links to B
- B links to A and C
- C links to A
- D links to A, B, and E
- E links to A and D

**First iteration (assuming PR = 1 for all):**

| Page | Calculation | Result |
|------|-------------|--------|
| PR(A) | (1-0.85) + 0.85(PR(B)÷2 + PR(C)÷1 + PR(D)÷3 + PR(E)÷2) | 1.878 |
| PR(B) | (1-0.85) + 0.85(PR(A)÷1) | 1.746 |
| PR(C) | (1-0.85) + 0.85(PR(B)÷2 + PR(D)÷3) | 1.175 |
| PR(D) | (1-0.85) + 0.85(PR(E)÷2) | 0.575 |
| PR(E) | (1-0.85) + 0.85(PR(D)÷3) | 0.433 |

Values are recalculated in subsequent iterations until they converge.

### Modern Search Engines

Modern search engines use **more complex algorithms** considering additional factors:
- **Age** of the web page
- **Loading speed** of content
- Content quality and relevance
- User engagement metrics

## Server-Side and Client-Side Processing

### Server-Side Processing

**Server-side processing** occurs when the **client sends data to a server** for processing. No processing happens on the client device.

**Common server-side languages:** SQL, PHP, Python, Node.js

**Advantages:**
- **No plugins required** on client
- Can perform **large calculations faster** than client devices
- **Not browser dependent** - works consistently across browsers
- **More secure** - code and data remain on server

### Client-Side Processing

**Client-side processing** occurs when data is processed on the **local device** (client computer). Also called **client-side scripting**.

**Common client-side language:** JavaScript

**Advantages:**
- **Immediate response** to user actions (no server round-trip)
- **Executes quickly** for simple operations
- **Greater control** over website behaviour and appearance
- Reduces server load

### Comparison

| Aspect | Server-Side | Client-Side |
|--------|-------------|-------------|
| Processing location | Remote server | Local device |
| Languages | PHP, SQL, Python | JavaScript |
| Security | More secure | Less secure (code visible) |
| Speed for simple tasks | Slower (network delay) | Faster |
| Speed for complex tasks | Faster (powerful hardware) | Slower |
| Browser dependency | Independent | Dependent |
