# 3.4 Web Technologies

## HTML

**HTML (HyperText Markup Language)** is the language used to describe the **structure and content** of web pages. Browsers **interpret and render** HTML to display webpages.

HTML uses **tags** written in angle brackets: `<tag>` (opening) and `</tag>` (closing).

### Document Structure

Every HTML document has two main sections:

- **Head** (`<head>`) - contains metadata and the page title; not displayed in the main browser window
- **Body** (`<body>`) - contains all visible content of the webpage

Basic structure:
```html
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    Content here
  </body>
</html>
```

### Common HTML Tags

| Tag | Purpose |
|-----|---------|
| `<html>` | Root element; all HTML code goes within these tags |
| `<head>` | Contains metadata and title |
| `<title>` | Sets the browser tab/window title |
| `<body>` | Contains visible page content |
| `<h1>` to `<h6>` | Headings in decreasing sizes |
| `<p>` | Paragraph (adds line spacing above and below) |
| `<a href="URL">text</a>` | Anchor tag for hyperlinks |
| `<img src="file" width="x" height="y">` | Image (self-closing tag) |
| `<ol>` | Ordered (numbered) list |
| `<ul>` | Unordered (bulleted) list |
| `<li>` | List item (used inside `<ol>` or `<ul>`) |
| `<div>` | Division - separates page into named areas |
| `<link>` | Links external files (e.g., CSS stylesheets) |

### Classes and Identifiers

**Classes** and **identifiers (IDs)** are attributes used to target elements for styling.

**Class** (`.`):
- Defined in CSS with a **full stop/period**: `.classname { }`
- Applied in HTML with: `class="classname"`
- **Multiple elements** can share the same class
- Used for consistent styling across elements

**Identifier/ID** (`#`):
- Defined in CSS with a **hash/hashtag**: `#idname { }`
- Applied in HTML with: `id="idname"`
- **Unique** - only one element per page can have a specific ID
- Used for targeting a single specific element

```css
/* Class - can be reused */
.highlight {
    background-color: yellow;
}

/* ID - unique element only */
#header {
    font-size: 24px;
}
```

```html
<p class="highlight">This uses a class</p>
<p class="highlight">This also uses the same class</p>
<div id="header">This uses a unique ID</div>
```

## CSS

**Cascading Style Sheets (CSS)** is a language used to describe the **visual appearance and styling** of HTML elements.

### Internal vs External CSS

**Internal/Embedded CSS**: Written directly in the HTML document within `<style>` tags in the head section.

**External CSS**: Written in a separate `.css` file and linked to the HTML document:
```html
<link href="styles.css" rel="stylesheet" type="text/css">
```

External CSS is preferred for larger projects as it:
- Separates content from presentation
- Allows one stylesheet to style multiple pages
- Makes maintenance easier

### CSS Syntax

```css
selector {
    property: value;
    property: value;
}
```

Example:
```css
body {
    margin: 0px;
    padding: 0px;
    background-color: white;
    font-family: Arial, Helvetica, sans-serif;
    font-size: 18px;
    text-align: center;
}
```

Each CSS rule consists of:
- **Selector** - the element, class, or ID to style
- **Declaration block** - curly braces containing property-value pairs
- **Properties** - aspects to style (e.g., `color`, `font-size`)
- **Values** - settings for each property

## JavaScript

**JavaScript** is a **scripting language** that adds **interactivity** to websites.

### Key Characteristics

- **Interpreted** (not compiled) - code is executed line by line by the browser each time the page loads
- **Platform independent** - runs on any processor architecture via the browser
- **Multi-paradigm** language - supports procedural, object-oriented, and functional programming styles
- Commonly used for **client-side input validation**

### Advantages of JavaScript

- **Local validation** - invalid data is caught on the client before being sent to servers
- **Reduces server load** - processing happens on user's device
- **Reduces network traffic** - fewer requests sent to server
- **Immediate feedback** - users see results instantly without page reload

### JavaScript Outputs

**Changing HTML element content:**
```javascript
chosenElement = document.getElementById("example");
chosenElement.innerHTML = "Hello World";
```

**Writing directly to the document:**
```javascript
document.write("Hello World");
```

**Displaying an alert box:**
```javascript
alert("Hello World");
```

## Search Engine Indexing

### Search Engines

A **search engine** is a program that searches through a **database of internet addresses** looking for resources based on user-specified criteria.

The **order of results** is critical as it determines which websites users visit.

### Web Crawlers

**Web crawlers** (also called **spiders**) are programs that collect information about websites to build the search engine's **index**.

How crawlers work:
1. **Traverse** the Internet by following **links** from page to page
2. Collect **keywords and phrases** from each page
3. Collect **metadata** (information specified by website owners)
4. Add all collected information to the search engine's **index**

### The Index

The **index** is a structured database containing:
- Keywords and phrases from web pages
- Metadata from websites
- Information about page structure and content

When users search, the engine queries this index rather than the entire web, enabling fast retrieval of relevant results.

## PageRank Algorithm

The **PageRank algorithm** determines the **ranking** of web pages in search results. Higher-ranked pages appear first.

### Factors Determining PageRank

1. **Number of incoming links** from other web pages
2. **PageRank of linking pages** - links from high-ranked pages carry more weight

### Data Structure

PageRank uses a **directed graph**:
- **Nodes** represent web pages
- **Arcs** (directed edges) represent links between pages

### The PageRank Formula

$$PR(x) = (1-d) + d \left[ \frac{PR(T_1)}{C(T_1)} + \frac{PR(T_2)}{C(T_2)} + \ldots + \frac{PR(T_n)}{C(T_n)} \right]$$

Where:
- **PR(x)** = PageRank of page x
- **PR(Tₙ)** = PageRank of page n that links to x
- **C(Tₙ)** = Count of **outbound links** from page n
- **d** = **Damping factor** (typically **0.85**)

### Damping Factor

The damping factor (d) represents the probability a user continues clicking links rather than navigating to a random page. Set to 0.85, meaning:
- 85% chance of following a link
- 15% chance of jumping to a random page

### Iterative Calculation

PageRank is calculated iteratively:
1. **First iteration**: Assume all pages have PageRank = 1
2. **Subsequent iterations**: Recalculate using values from previous iteration
3. **Continue** until values stabilise (converge)

### Worked Example

For a graph with pages A, B, C, D, E where:
- A links to B
- B links to A and C
- C links to A
- D links to A, B, and E
- E links to A and D

**First iteration** (assuming PR = 1 for all):

| Page | Formula | Result |
|------|---------|--------|
| PR(A) | (1-0.85) + 0.85(PR(B)/2 + PR(C)/1 + PR(D)/3 + PR(E)/2) | 1.878 |
| PR(B) | (1-0.85) + 0.85(PR(A)/1) | 1.746 |
| PR(C) | (1-0.85) + 0.85(PR(B)/2 + PR(D)/3) | 1.175 |
| PR(D) | (1-0.85) + 0.85(PR(E)/2) | 0.575 |
| PR(E) | (1-0.85) + 0.85(PR(D)/3) | 0.433 |

Subsequent iterations use the calculated values, with rankings converging over multiple iterations.

### Modern Search Engines

Real search engines use **more complex algorithms** considering additional factors:
- Page age
- Content loading speed
- Mobile-friendliness
- Content relevance and quality
- User engagement metrics

## Server and Client Side Processing

### Server Side Processing

**Server side processing** occurs when the **client sends data to a server** for processing. No processing happens on the client device.

**Common server-side languages**: SQL, PHP, Python, Ruby, Java

**Advantages:**
- **No plugins required** on client
- Can perform **large calculations faster** than clients
- **Not browser dependent** - works regardless of client browser
- **More secure** - code runs on controlled server environment
- Better for handling **sensitive data** and database operations

### Client Side Processing

**Client side processing** (or **client side scripting**) occurs when data is processed on the **user's local device**.

**Common client-side language**: JavaScript

**Advantages:**
- **Immediate response** to user actions (no server round-trip)
- **Executes quickly** - no network delay
- **Greater control** over website behaviour and appearance
- **Reduces server load** - processing distributed across clients
- Works **offline** for some functionality

### Comparison

| Aspect | Server Side | Client Side |
|--------|-------------|-------------|
| **Processing location** | Remote server | User's device |
| **Speed** | Slower (network latency) | Faster (local) |
| **Security** | More secure | Less secure (code visible) |
| **Server load** | Higher | Lower |
| **Browser dependency** | Independent | Dependent |
| **Offline capability** | Requires connection | Some functionality possible |
| **Validation** | Final validation | Initial validation |

### Typical Use Cases

**Server side**: Database operations, user authentication, complex calculations, secure transactions

**Client side**: Form validation, UI interactions, animations, immediate feedback
