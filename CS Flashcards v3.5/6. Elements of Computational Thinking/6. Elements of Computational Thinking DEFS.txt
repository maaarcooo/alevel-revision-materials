What is abstraction? | Simplifying a problem by only considering the necessary details required for a solution, removing all unnecessary information
Simplifying a problem by removing unnecessary details and keeping only what is needed for a solution — what is this called? | Abstraction
What is data abstraction? | The storage and representation of data in a computer system along with its logical description and interaction with operators (e.g. stacks, queues, integers)
The representation of data in a computer system along with its logical description and how it interacts with operators — what term describes this? | Data abstraction
Give three examples of data abstraction | Stacks, queues, integers
What is graph theory? | A branch of mathematics used to abstractly represent problems using a collection of nodes connected by edges
A branch of mathematics that represents problems using nodes connected by edges — what is this called? | Graph theory
What two components make up a graph in graph theory? | Nodes and edges
Why is graph theory useful in computational thinking? | It allows complex real-world problems to be represented abstractly, making them easier to model and solve computationally
How does abstraction relate to problem-solving in computing? | It reduces complexity by stripping away irrelevant details, allowing the programmer to focus only on the information needed to solve the problem
What is caching? | The temporary storage of data by a system in cache or memory so it can be retrieved faster if needed again in future
The temporary storage of data for faster future retrieval — what is this called? | Caching
Why is caching beneficial? | It speeds up data retrieval because frequently accessed data does not need to be fetched from the original (slower) source each time
What are inputs in the context of computational thinking? | Any information relevant to the problem that is required by the system for processing according to an algorithm
What are outputs in the context of computational thinking? | The result returned by a system for a given input after running all or part of a process
How do inputs and outputs relate to each other in a system? | Inputs are the data fed into a system for processing by an algorithm, and outputs are the results produced after that processing
What are preconditions? | Prerequisites or states of a system and its surroundings that must be met before a use case can run and return a valid solution
Requirements that must be satisfied before a process can execute correctly — what term describes these? | Preconditions
Why is it useful to define preconditions? | They clarify what must be true before a function or program runs, helping users understand requirements and reducing the need for unnecessary validation checks in code
What are reusable program components? | Components that have already been written, debugged, and tested, which can be transplanted into new systems to save development time
Pre-written, debugged, and tested components that can be used in new systems — what are these called? | Reusable program components
Why are reusable program components beneficial? | They save development time because the components have already been written, debugged, and tested, reducing effort and improving reliability
What is procedural abstraction? | Achieving a task by relying on a procedure of sequential steps
Achieving a task by following a defined sequence of steps — what term describes this? | Procedural abstraction
What are the components of a problem in procedural thinking? | The smaller, simpler series of tasks and sub-procedures that a problem can be broken down into to be completed modularly
Breaking a problem into smaller, simpler sub-tasks that can be completed modularly — what is this process identifying? | The components of a problem
Why is decomposing a problem into components useful? | It makes complex problems more manageable by dividing them into smaller, simpler tasks that can be developed and tested independently
What is a flowchart? | A diagrammatic representation of the flow of a program that includes all the decision points needed to obtain a solution
A diagrammatic representation showing program flow and all decision points — what is this called? | A flowchart
What are logical conditions? | Conditions depending on one or more variables that determine the next step when a system must make a decision
Conditions based on variables that determine the next step at a decision point — what term describes these? | Logical conditions
Name three control structures used to implement logical conditions | Sequences, selections, and iterations
What is concurrent processing? | Processing where a system appears to perform multiple tasks simultaneously
Processing where a system appears to carry out multiple tasks at the same time — what is this called? | Concurrent processing
How does abstraction differ from procedural abstraction? | Abstraction removes unnecessary detail from a problem to simplify it. Procedural abstraction specifically focuses on breaking a solution into a sequence of ordered steps to achieve a task
How do inputs differ from preconditions? | Inputs are the data fed into a system for processing. Preconditions are the states or requirements that must be true before the system can run correctly — they are not data to process but conditions to satisfy
How does caching relate to thinking ahead? | Caching anticipates that data will be needed again in future and stores it temporarily for faster retrieval, which is a forward-planning strategy to improve performance
