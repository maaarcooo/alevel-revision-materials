What is an operating system? | A collection of programs that provide an interface between the user and computer, managing hardware and software resources
A collection of programs that provide an interface between the user and computer, managing hardware and software resources — what term is this? | Operating system
What two types of user interface can an operating system provide? | Graphical user interface (GUI) and command-line interface (CLI)
Name three examples of utility software provided by an operating system | Disk defragmenter, backup software, file encryption
What does disk defragmentation do? | Reorganises files on the hard disk so they are stored contiguously, allowing them to be found and accessed more quickly
What is paging? | A memory management technique that divides memory into equal-sized blocks called pages, which can be swapped between main memory and the hard disk as needed
A memory management technique that divides memory into equal-sized blocks that can be swapped between main memory and the hard disk — what is this? | Paging
What is segmentation? | A memory management technique that divides memory into variable-sized logical divisions called segments, corresponding to the structure of the program (e.g. loops, conditional statements)
A memory management technique that divides memory into variable-sized logical divisions reflecting the structure of a program — what is this? | Segmentation
What is virtual memory? | A section of the hard disk used as an extension of RAM when main memory is insufficient, allowing programs not currently in use to be temporarily moved out of RAM
A section of the hard disk used as an extension of RAM when main memory is full — what is this called? | Virtual memory
What is disk thrashing? | When the computer freezes because pages are being swapped too frequently between the hard disk and main memory, so more time is spent transferring pages than running programs. It worsens as virtual memory fills up
What is internal fragmentation? | Wasted space within a fixed-size page when the allocated data does not completely fill the block
What is external fragmentation? | Small gaps of unusable free space left between variable-sized segments in memory as segments are allocated and deallocated over time
How does paging differ from segmentation? | Paging divides memory into equal-sized blocks (pages) and can cause internal fragmentation. Segmentation divides memory into variable-sized logical divisions (segments) and can cause external fragmentation
What is an interrupt? | A signal generated by software or hardware to indicate to the processor that a process needs attention
A signal generated by software or hardware indicating to the processor that a process needs attention — what is this called? | An interrupt
When does the processor check for interrupts during the FDE cycle? | At the end of each fetch-decode-execute cycle
Where are pending interrupts stored? | In a priority queue within the interrupt register, ordered by priority
What happens when an interrupt with higher priority than the current process is detected? | The current contents of the CPU's special-purpose registers are saved to a stack, the appropriate ISR is loaded into RAM, and a flag is set to signal the ISR has begun
What is an Interrupt Service Routine (ISR)? | A routine that handles a specific type of interrupt. The processor loads and executes the corresponding ISR when an interrupt needs servicing
What happens after an ISR completes? | The flag is reset and the interrupt queue is checked for further higher-priority interrupts. If none remain, register contents are restored from the stack and the FDE cycle resumes
What data structure is used to save CPU register contents when an interrupt is being serviced? | A stack
Name two examples of interrupts | A printer signalling completion of a print job, or a peripheral signalling a power failure
Name three types of interrupt | Hardware interrupts (from external devices, e.g. keyboard input), software interrupts (from programs or the OS, e.g. division by zero), and trap interrupts (intentionally triggered for debugging or error handling)
What is interrupt nesting? | The ability of the processor to pause a lower-priority ISR to handle a higher-priority interrupt that arrives during its execution
What is scheduling? | The process by which the operating system determines which jobs receive processor time, for how long, and in what order, using scheduling algorithms
What is pre-emptive scheduling? | A scheduling approach where the OS can actively stop a running job and reallocate the CPU, giving each process a time-limited slot
What is non-pre-emptive scheduling? | A scheduling approach where a job runs to completion once started, without being interrupted by the OS
What is a time slice (time quantum)? | A fixed period of processor time allocated to each job in pre-emptive scheduling algorithms such as Round Robin
What is processor starvation? | When a process does not receive enough processor time to execute and complete, typically because shorter or higher-priority jobs keep being scheduled ahead of it
A process not receiving enough processor time to execute and be completed — what is this called? | Processor starvation
How does Round Robin scheduling work? | A pre-emptive algorithm where each job receives an equal time slice. If incomplete after its slice, the job moves to the back of the queue. This cycles until all jobs complete
What is a disadvantage of Round Robin scheduling? | It does not account for job priority, and long jobs take much longer to complete because their execution is split across many cycles
How does First Come First Served (FCFS) scheduling work? | A non-pre-emptive algorithm where jobs are processed in the order they arrive in the queue, each running to completion before the next begins
What is a disadvantage of FCFS scheduling? | It does not allocate processor time based on priority, so short or high-priority jobs may wait behind long jobs
How does Multi-Level Feedback Queue (MLFQ) scheduling work? | A pre-emptive algorithm using multiple queues, each with a different priority. All processes start in the highest-priority queue and move down to lower-priority queues if they exceed their time quantum
What is a disadvantage of MLFQ scheduling? | Complex to implement due to the difficulty of configuring multiple queues and deciding how to prioritise jobs based on a combination of factors
How does Shortest Job First (SJF) scheduling work? | A non-pre-emptive algorithm where the queue is ordered by estimated time to completion, with the shortest jobs processed first
What is a disadvantage of SJF scheduling? | Requires knowing or estimating how long each job will take (not always possible), and longer jobs risk processor starvation if short jobs keep arriving
How does Shortest Remaining Time (SRT) scheduling work? | A pre-emptive version of SJF where the queue is reordered before each cycle by remaining time to completion. If a new job arrives with less remaining time, the current job is pre-empted
What is a disadvantage of SRT scheduling? | Longer jobs risk processor starvation if short jobs keep arriving, and frequent pre-emption causes high context-switching overhead
Which scheduling algorithms carry a risk of processor starvation? | Shortest Job First and Shortest Remaining Time, because shorter jobs are always given priority over longer ones
What is a distributed operating system? | An OS that runs across multiple devices, spreading the processing load across multiple computer processors when a task is run
What is an embedded operating system? | An OS built to perform a small range of specific tasks on a particular device (e.g. washing machine, microwave). Limited in functionality and hard to update, but consumes significantly less power
What is a multi-tasking operating system? | An OS that enables the user to carry out tasks seemingly simultaneously by using time slicing to switch quickly between programs in memory
What is a multi-user operating system? | An OS where multiple users share one computer, requiring a scheduling algorithm to ensure processor time is divided fairly. Without proper scheduling, processor starvation can occur
What is a real-time operating system? | An OS designed to process tasks within a guaranteed time frame, used in time-critical systems such as nuclear power station control or self-driving cars
What is the BIOS? | The Basic Input Output System. Firmware stored on the motherboard that is the first program to run when a computer is switched on
What does the BIOS do on startup? | Performs the POST (Power-On Self-Test), which checks that hardware such as keyboards, disk drives, memory, and the CPU is correctly connected and functional
What happens after the BIOS completes POST successfully? | It runs the bootstrap loader, which loads the operating system from the hard disk into RAM
What is a bootstrap loader (bootloader)? | The program that loads the operating system from the hard disk into main memory after POST completes
Which CPU register points to the location of the BIOS on startup? | The Program Counter (PC)
What is a device driver? | Software that enables the operating system to communicate with and control a specific hardware device
Software that enables the operating system to interact with and control a hardware device — what is this called? | A device driver
Why are device drivers specific to both architecture and operating system? | Different hardware architectures (e.g. smartphone vs desktop) and different operating systems require different communication protocols, so each driver must be written for a specific architecture-OS combination
What is a virtual machine? | A software implementation of a computer system that provides an environment for running programs, including executing intermediate code via a built-in translator
A software implementation of a computer system that provides an environment with a translator for intermediate code — what is this called? | A virtual machine
What is intermediate code? | Platform-independent code produced by compilation that sits between source code and machine code, executable on any system with an appropriate virtual machine (e.g. Java bytecode on the JVM)
What software manages multiple virtual machines running on a single physical machine? | Hypervisor software, which allocates physical hardware resources to each VM
Name two uses of virtual machines for software development | Testing programs across different operating systems without purchasing extra hardware, and providing isolated environments where malware or risky code cannot affect the host system
What is an advantage of executing intermediate code via a virtual machine? | The same code can run on different machines and operating systems (platform independence), saving development time and cost
What is a disadvantage of running programs in a virtual machine? | Execution is considerably slower than running native low-level code on the physical hardware, and VMs consume host system resources (CPU, RAM, storage)
