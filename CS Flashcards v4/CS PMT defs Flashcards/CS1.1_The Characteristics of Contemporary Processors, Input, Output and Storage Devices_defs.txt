What are the three stages of the Fetch-Decode-Execute cycle? | Fetch (supply address and retrieve instruction from memory), Decode (interpret instruction and retrieve required data), Execute (CPU carries out the required actions)
Explain the Fetch stage of the FDE cycle | The address of the next instruction is supplied and the instruction is retrieved from memory
Explain the Decode stage of the FDE cycle | The instruction is interpreted and the required data is read and retrieved from their addresses
Explain the Execute stage of the FDE cycle | The CPU carries out the required actions specified by the instruction
What is Harvard Architecture? | A computer architecture that stores data and instructions in separate memories, allowing the next instruction to be read while data is currently being read or written
A computer architecture with separate memories for data and instructions, enabling simultaneous data and instruction access — what is this called? | Harvard Architecture
Why does Harvard Architecture allow faster processing than Von Neumann? | Data and instructions are stored in separate memories, so the next instruction can be fetched while data is still being read or written, avoiding the Von Neumann bottleneck

What is the Memory Data Register (MDR)? | A special register that temporarily stores data to be read from or written to the computer's memory
A register that temporarily holds data being transferred to or from main memory — what is this called? | The Memory Data Register (MDR)
What is a core in a processor? | A processing unit that handles instructions with its own fetch-decode-execute cycle
How do multi-core processors improve performance? | They have multiple cores that can run simultaneously, allowing multiple instructions to be processed at the same time
What is pipelining? | The simultaneous processing of several instructions by decoding the next instruction and fetching the one after while the current one is being executed
Why does pipelining improve CPU throughput? | Different stages of the FDE cycle are handled concurrently for different instructions, so the CPU completes more instructions per unit time
What is the Program Counter (PC)? | A special purpose register that stores the address of the next instruction to execute
A register that holds the address of the next instruction to be executed — what is this called? | The Program Counter (PC)
What are registers? | Special memory cells that can be accessed very quickly, used to temporarily store data and control information

What is Von Neumann Architecture? | A computer architecture where a single control unit manages program control via a linear sequence of fetch-decode-execute cycles
A computer architecture using a single control unit with a linear sequence of FDE cycles — what is this called? | Von Neumann Architecture
How does Von Neumann Architecture differ from Harvard Architecture? | Von Neumann stores data and instructions in the same memory (shared bus), causing a bottleneck. Harvard stores them in separate memories, allowing simultaneous access to data and instructions
How does the Address Bus differ from the Data Bus? | The Address Bus is unidirectional and carries memory addresses. The Data Bus is bi-directional and carries data and instructions
How does the MAR differ from the MDR? | The MAR stores the memory address of the next instruction or data location. The MDR stores the actual data being read from or written to memory
How does the Program Counter (PC) differ from the Current Instruction Register (CIR)? | The PC stores the address of the next instruction to execute. The CIR stores the instruction currently being decoded and executed
What is a Complex Instruction Set Computer (CISC)? | A processor design that can execute a series of tasks in a single complex instruction built into the hardware, using less RAM but preventing pipelining
A processor design using complex multi-task instructions built into hardware, reducing RAM usage but preventing pipelining — what is this? | CISC (Complex Instruction Set Computer)
What is a Reduced Instruction Set Computer (RISC)? | A simpler processor design that executes a single simple instruction each clock cycle, using more RAM but allowing pipelining
A processor design using simple single-cycle instructions that enable pipelining but require more RAM — what is this? | RISC (Reduced Instruction Set Computer)
How does CISC differ from RISC? | CISC uses complex multi-task instructions (less RAM, no pipelining). RISC uses simple single-cycle instructions (more RAM, supports pipelining)
Why can RISC processors use pipelining but CISC processors cannot? | RISC instructions each take one clock cycle and have a uniform length, making it easy to overlap stages. CISC instructions vary in length and complexity, making pipeline stages uneven

A specialised processor with many small cores designed for efficient parallel computation — what is this called? | A GPU (Graphic Processing Unit)
