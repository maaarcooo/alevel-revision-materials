What is an operating system? | A collection of programs that manages the hardware and software resources of a computer and provides an interface between the user and the hardware
A collection of programs that manages hardware/software resources and provides an interface between the user and the computer — what is this? | An operating system
Name the seven main functions of an operating system | Resource management, file management, interrupt handling, security, providing a platform for software, providing a user interface, providing utilities
What does resource management by the OS involve? | Managing the CPU, memory, disk drives, and printers — allocating resources to specific tasks and ensuring they are used effectively
When a user opens multiple applications simultaneously, what three decisions does the OS make? | How much memory to allocate to each application, when and for how long each gets to use the CPU, and how to handle data being read from or written to the hard drive
What does file management by the OS involve? | Handling the storage, retrieval, and manipulation of data files, including providing a GUI of the file system for naming files and choosing directories
Explain the role of the OS in interrupt handling | The OS handles interrupt events that require immediate CPU attention, processing them in a timely manner to maintain smooth system operation
Name four security features provided by an operating system | Password-protected system accounts, firewall, virus scanning, and file encryption
How does the OS provide a platform for software to run? | By allowing application software access to system resources such as the GPU, network card, memory, and CPU
What two types of user interface can an operating system provide? | A graphical user interface (GUI) for visual interaction, and a command-line interface (CLI) for text-based interaction
What is the purpose of utility programs in an OS? | To help with system maintenance and security tasks
Name five examples of utility software | File encryption, file compression, disk defragmentation, system backup, disk cleanup
What does disk defragmentation do? | Physically reorganises files on the hard disk so they can be found and accessed faster
What does disk cleanup do? | Scans the hard disk for duplicate and corrupt files and deletes them to free up disk space
What does file compression do? | Reduces the size of a file, helping send large files over a network or saving disk space
What does backup software do? | Allows users to create system snapshots and restore their system to a previous point in history
What is memory management? | A fundamental OS role dealing with the allocation and deallocation of the computer's primary memory
The allocation and deallocation of primary memory by the OS — what function is this? | Memory management
When a user opens an application, what happens to its data? | It is loaded from storage into active (primary) memory so it can run smoothly
Name two benefits of memory management | Efficient allocation enables multitasking (multiple programs running at once), and it maintains security by preventing programs from accessing memory reserved for other programs
Name the three techniques that make memory management more efficient | Paging, segmentation, and virtual memory
What is paging? | A method of dividing primary memory into equal-sized blocks called pages, which can be dynamically allocated and deallocated to applications
A method of dividing memory into equal-sized blocks — what memory management technique is this? | Paging
When an application is launched, what happens to its data in paging? | Data is moved from the hard disk into pages in primary memory for faster access
Explain why paging can lead to internal fragmentation | If data does not perfectly fill a page, the unused space in the last page is wasted because other unrelated data cannot be stored in that page. Over time, these pockets of wasted space accumulate
What is internal fragmentation? | Wasted space within fixed-size memory pages caused by data not completely filling the allocated page
Wasted space inside fixed-size memory blocks because data does not fill them completely — what is this called? | Internal fragmentation
What is segmentation? | A method of dividing memory into variable-sized blocks (segments) that correspond to different logical types of data needed by an application
A method of dividing memory into variable-sized blocks based on logical data types — what memory management technique is this? | Segmentation
Give an example of how segmentation works in a video editing application | Different segments may be created for video data, audio data, special effects, and UI elements — each sized according to its data requirements
Why is segmentation considered space-efficient? | Because it only allocates space depending on the amount an application actually needs, rather than using fixed-size blocks
Explain why segmentation can lead to external fragmentation | As segments of varying sizes fill up and are removed from memory, physical gaps appear that may be too small for new segments, reducing the maximum size of new segments that can be allocated
What is external fragmentation? | Physical gaps between allocated memory segments that reduce the maximum contiguous space available for new segments
Wasted gaps between variable-sized memory blocks that limit the space available for new allocations — what is this called? | External fragmentation
How does internal fragmentation differ from external fragmentation? | Internal fragmentation is wasted space within fixed-size pages (caused by paging). External fragmentation is wasted gaps between variable-sized segments (caused by segmentation)
What is virtual memory? | A technique where the OS uses a section of secondary storage (hard drive/SSD) to act as an extension of primary memory when RAM is insufficient
A technique where secondary storage acts as an extension of RAM — what is this called? | Virtual memory
Explain how virtual memory enables continued multitasking | The OS offloads less frequently used data from RAM into virtual memory on the hard drive, freeing up RAM for programs that need immediate access
Why is virtual memory slower than RAM? | Because accessing data on secondary storage (HDD or SSD) is considerably slower than accessing data in RAM, even though SSDs are faster than HDDs
What is disk thrashing? | When the computer freezes because pages are being swapped too frequently between the hard disk and main memory, spending more time transferring pages than running programs
Explain why over-reliance on virtual memory leads to performance issues | Data must be read from and written to secondary storage, which is much slower than RAM. Excessive swapping (disk thrashing) means the system spends more time moving data than processing it
What is an interrupt? | A signal to the processor that stops its current task and requires it to perform a different task temporarily
A signal that causes the processor to suspend its current task and handle a different task — what is this? | An interrupt
Name three purposes of interrupts | Real-time event handling (e.g. hardware errors), device communication (e.g. printer jams), and multitasking (suspending one application to switch to another)
What are hardware interrupts? | Interrupts generated by external devices, such as keyboard input, mouse movements, or disk I/O requests
Interrupts generated by external devices like keyboards or mice — what type of interrupt is this? | Hardware interrupts
What are software interrupts? | Interrupts triggered by software or the operating system, such as application requests to open a file or division by zero errors
Interrupts triggered by programs or the OS, such as division by zero errors — what type of interrupt is this? | Software interrupts
What are trap interrupts? | Interrupts intentionally triggered by a program, used for software debugging and handling unexpected error cases
Interrupts intentionally triggered by a program for debugging purposes — what type of interrupt is this? | Trap interrupts
What happens during the Interrupt Request (IRQ) stage? | An external device or software generates an interrupt signal, which is passed by the interrupt controller to the interrupt handler for assessment
What happens during the Interrupt Acknowledge stage? | The interrupt handler decides if the interrupt needs to be dealt with now or later. If now, the current contents of the processor registers are saved in memory (onto a stack)
What happens during the ISR Lookup stage? | The processor fetches the corresponding Interrupt Service Routine associated with the interrupt type
What happens during ISR Execution? | The processor transfers control to the ISR and executes the routine to handle the specific interrupt
What happens during the Interrupt Exit stage? | The processor restores the saved register contents from the stack and the fetch-decode-execute cycle resumes
What is an Interrupt Service Routine (ISR)? | A special function that handles a particular interrupt type, designed to be concise and efficient to minimise execution time for time-sensitive events
A special function designed to handle a specific type of interrupt — what is this called? | An Interrupt Service Routine (ISR)
At what point in the CPU cycle are interrupts checked? | At the end of each fetch-decode-execute cycle
Where are the register contents saved when an interrupt is serviced? | Onto a stack (a LIFO data structure)
What is interrupt prioritisation? | The ability of the processor to acknowledge and switch to resolving a higher-priority interrupt, suspending lower-priority ISRs until the higher-priority one completes
What is nesting of interrupts? | The ability of the processor to handle interrupts within interrupts — a higher-priority interrupt can interrupt a lower-priority ISR that is currently executing
Why is proper management of nested interrupts important? | To avoid potential conflicts between interrupt routines and ensure system stability
Where are interrupts stored while waiting to be serviced? | In an interrupt register, ordered by priority in a priority queue
What is scheduling? | The process of deciding which tasks to process, for how long, and in what order, using scheduling algorithms to allocate CPU time
Deciding which tasks get CPU time, for how long, and in what order — what OS function is this? | Scheduling
What is pre-emptive scheduling? | Scheduling that allocates the CPU for time-limited slots (time quanta), allowing processes to be interrupted before completion
Scheduling that allocates CPU time in limited slots and allows interruption of running processes — what category is this? | Pre-emptive scheduling
What is non-pre-emptive scheduling? | Scheduling that allocates the CPU to a process until it completes its burst time or switches to a waiting state, without interruption
Scheduling where a process holds the CPU until it finishes or enters a waiting state — what category is this? | Non-pre-emptive scheduling
How does Round Robin scheduling work? | Each process is given an equal time quantum. If a process is not completed within its quantum, it moves to the back of the queue. This cycles until all processes complete
Is Round Robin pre-emptive or non-pre-emptive? | Pre-emptive
Name one benefit and one drawback of Round Robin | Benefit: all processes get a fair share of CPU time. Drawback: choosing the right time quantum is difficult, and long processes have high turnaround times
How does First Come First Served (FCFS) scheduling work? | Processes are served in the order they arrive. The current process blocks all others until it completes. New tasks join the back of the queue
Is FCFS pre-emptive or non-pre-emptive? | Non-pre-emptive
Name one benefit and one drawback of FCFS | Benefit: simple and easy to implement. Drawback: poor performance if a long process arrives before shorter ones (no priority handling)
How does Multi-Level Feedback Queue (MLFQ) scheduling work? | Multiple queues with different priorities are used. All processes start in the highest-priority queue but trickle down to lower-priority queues if they exceed their time quantum
Is MLFQ pre-emptive or non-pre-emptive? | Pre-emptive
Name one benefit and one drawback of MLFQ | Benefit: smaller/shorter tasks are prioritised efficiently. Drawback: more complex to implement and requires careful parameter tuning (number of queues, ageing rules)
How does Shortest Job First (SJF) scheduling work? | All processes are continuously sorted by burst time from shortest to longest. Shorter jobs are placed at the front and processed first
Is SJF pre-emptive or non-pre-emptive? | Non-pre-emptive
Name one benefit and one drawback of SJF | Benefit: minimises average waiting time. Drawback: requires knowing burst times in advance, and long processes can starve if short ones keep arriving
How does Shortest Remaining Time First (SRTF) scheduling work? | A pre-emptive version of SJF where processes are ordered by the shortest remaining time to complete. After each time quantum, all processes are re-inspected and reordered
Is SRTF pre-emptive or non-pre-emptive? | Pre-emptive
Name one benefit and one drawback of SRTF | Benefit: ideal for prioritising jobs with shorter burst times. Drawback: high context-switching overhead and risk of starvation for longer jobs
What is processor starvation? | When a particular process does not receive enough processor time to execute and be completed, typically because shorter or higher-priority processes keep arriving
A process not receiving enough CPU time to complete — what is this called? | Processor starvation
Which scheduling algorithms carry a risk of processor starvation? | Shortest Job First (SJF) and Shortest Remaining Time First (SRTF) — long jobs can starve if short jobs keep arriving
How do pre-emptive and non-pre-emptive scheduling differ? | Pre-emptive scheduling can interrupt a running process when its time quantum expires or a higher-priority process arrives. Non-pre-emptive scheduling lets a process run until it finishes or voluntarily yields
Name two pre-emptive and two non-pre-emptive scheduling algorithms | Pre-emptive: Round Robin, SRTF (also MLFQ). Non-pre-emptive: FCFS, SJF
What is a distributed operating system? | An OS that runs on multiple machines appearing as a single unit, used for efficient task distribution and load balancing across multiple processors
An OS that runs across multiple devices, spreading the processing load — what type of OS is this? | Distributed operating system
What is an embedded operating system? | An OS designed for specific tasks, running inside a device that is not primarily a computer (e.g. microwave, washing machine), with limited functionality and low power consumption
An OS built for a specific device with a small range of functions and low power consumption — what type of OS is this? | Embedded operating system
What is a multi-tasking operating system? | An OS that allows multiple tasks to run concurrently on a single processor by using time slicing to switch quickly between programs in memory
An OS that allows multiple applications to appear to run simultaneously using time slicing — what type of OS is this? | Multi-tasking operating system
What is a multi-user operating system? | An OS that supports multiple users accessing computer resources concurrently, efficiently managing resource allocation with scheduling and providing data security and user privacy
An OS where multiple users share one computer, requiring scheduling to share processor time fairly — what type of OS is this? | Multi-user operating system
What is a real-time operating system (RTOS)? | An OS designed for immediate data processing that guarantees tasks are completed within specific timeframes, used where low latency is critical to safety
An OS designed to guarantee task completion within strict timeframes — what type of OS is this? | Real-time operating system (RTOS)
Give two examples of where a real-time OS would be used | Aerospace systems (e.g. control rods at a nuclear power station) and automotive systems (e.g. self-driving cars) — anywhere a guaranteed response time is critical to safety
What does BIOS stand for? | Basic Input/Output System
What is the BIOS? | Firmware stored on a small memory chip on the motherboard that is the first software to run when a computer starts up
Firmware on the motherboard that is the first software to run on startup — what is this? | BIOS (Basic Input/Output System)
What does BIOS do on system startup? | It performs a POST (Power-On Self-Test) to check all hardware components are working, then runs the bootstrap loading sequence to start the operating system
What is POST? | Power-On Self-Test — a diagnostic testing sequence run by the BIOS that ensures all hardware components are connected and functioning correctly
A diagnostic sequence that checks hardware is working correctly on startup — what is this called? | POST (Power-On Self-Test)
What is the bootstrap loading sequence? | The program run by the BIOS (after a successful POST) that is responsible for loading the operating system from the hard disk into RAM
The program responsible for loading the OS from the hard disk into RAM after POST succeeds — what is this called? | Bootstrap / bootloader
What happens if the BIOS is overwritten or destroyed? | The computer cannot perform POST or load the bootstrap program, so the operating system will fail to start and the computer will be unusable
What is a device driver? | Software that enables communication between an operating system and a specific hardware device, allowing the OS to control and interact with that device
Software that enables the OS to communicate with and control a specific hardware device — what is this called? | A device driver
Why are device drivers needed? | Because external hardware devices have their own embedded system software, so a driver bridges the gap between the main OS and the device's hardware OS, enabling specific operations
Give three examples of hardware that requires device drivers | Printers, graphics cards, and network cards
Why might an OS have several drivers for the same type of device (e.g. multiple printer drivers)? | Because most hardware manufacturers write their own driver software, so different brands/models of the same device type each need their own driver
Are device drivers specific to the OS, the hardware architecture, or both? | Both — drivers are specific to the computer's architecture (device type) and to the operating system installed on the device
What is a virtual machine (VM)? | A software-driven computer that runs within a physical machine, mimicking a complete computer system including virtual CPU, memory, storage, and network interface
A software implementation of a complete computer system running within a physical machine — what is this? | A virtual machine (VM)
What is hypervisor software? | Software that enables a physical machine to run multiple virtual machines, managing resource allocation between the host OS and VMs
Software that allows multiple VMs to run on a single physical machine — what is this called? | Hypervisor software
What is intermediate code (bytecode)? | Code that is halfway between source code and machine code, independent of processor architecture, so it can run on different machines and operating systems via a virtual machine
Code compiled to a platform-independent form that runs on a virtual machine rather than directly on hardware — what is this called? | Intermediate code (bytecode)
Give an example of intermediate code execution | Java compiles to bytecode, which is then executed by the Java Virtual Machine (JVM), allowing the same program to run on multiple platforms
Name three uses of virtual machines | Cross-platform compatibility (run software from different OSs), software testing in isolated environments, and legacy support (running older OS versions for outdated applications)
Explain why VMs are useful for software testing | They create isolated environments separate from the host OS, so developers can safely test software (including malware), monitor system performance, and test on clean-slate systems without affecting their main machine
How can VMs help with cross-platform compatibility? | A user can run a VM of a different OS (e.g. macOS on Windows) to use software designed exclusively for that OS, without needing separate physical hardware
What is forwards compatibility in the context of VMs? | Running a VM of a previous OS version to use applications that have not been updated to work with the latest OS release
Name two drawbacks of using virtual machines | VMs share the host machine's hardware resources (CPU, RAM, storage) so over-use can exhaust the host system. Performance is usually slower than running software directly on physical hardware
What happens if a VM is allocated too many system resources? | The host OS may experience crashes, lag, or performance degradation due to insufficient remaining resources
How does a VM help protect against malware during development? | Malware tested or encountered in the VM affects only the virtual environment, leaving the host operating system unaffected. The VM can be easily reset using snapshots
Name one additional cost consideration when using VMs | Operating systems often require an activation payment or licence for full feature access, adding cost when running multiple virtualised OSs
