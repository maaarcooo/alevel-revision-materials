What is abstraction in computer science? | The process of removing unnecessary details from a problem to focus on the important features needed to implement a solution
The process of removing excessive details to arrive at a representation consisting of only the key features — what is this called? | Abstraction
What is representational abstraction? | Analysing what is relevant to a given scenario and simplifying a problem by removing unnecessary details based on this information
Simplifying a problem by analysing relevance and keeping only key features — what type of abstraction is this? | Representational abstraction
What is abstraction by generalisation? | Grouping together similarities within a problem to identify what kind of problem it is, so that a common solution can be applied to problems of that type
Grouping similarities within problems to categorise them and apply common solutions — what type of abstraction is this? | Abstraction by generalisation
What is data abstraction? | Hiding details about how data is stored, allowing programmers to use abstract data structures (e.g. stacks, queues) without knowing how they are implemented
Hiding details about how data is stored so programmers can use structures without knowing their implementation — what is this called? | Data abstraction
What is procedural abstraction? | Modelling what a subroutine does without considering how it achieves this, allowing the procedure to be reused as a black box
Modelling what a subroutine does without considering how it does it, so it can be reused as a black box — what term describes this? | Procedural abstraction
How does abstraction by generalisation differ from representational abstraction? | Representational abstraction simplifies a specific problem by removing unnecessary details. Abstraction by generalisation categorises problems by identifying shared features, allowing a common solution to be reused across similar problems
Give three examples of abstraction in real-world computing applications | A flight or car simulator, a map of a bus or train route, a representation of a building in a game or program
Why is the London Underground map a good example of abstraction? | It removes geographical layout details (unnecessary for travellers) and shows only stations and connections, which is all a traveller needs to navigate from one stop to another
In the context of abstraction, what does it mean when an implementation is "closer to reality"? | The solution includes more real-world details and fewer simplifications, making it less abstract
Why is Pong considered a highly abstracted game? | It models tennis/badminton but removes real-world factors like gravity, friction, air resistance, and variable ball momentum, keeping only the essential paddle-and-ball interaction
Why is abstraction needed in software development? | It allows programmers to focus on the essential elements of a solution rather than worrying about unnecessary details, reducing development time and preventing programs from becoming unnecessarily large
Explain how abstraction allows non-experts to use complex systems | Abstraction hides information that is too complex or irrelevant to the system's purpose, providing a simplified interface that does not require specialist knowledge to operate
How are maps abstracted so that computers can process routes? | Unnecessary details (buildings, greenspace) are removed; junctions become graph nodes, roads become edges/arcs, one-way rules become directed edges, and distances become edge weights
After a city map is fully abstracted into a graph, what algorithms can be applied to find shortest routes? | Dijkstra's shortest path algorithm and A* search algorithm
What additional real-world factors can apps like Google Maps incorporate beyond the basic abstracted graph? | Traffic conditions, roadworks, and weather conditions
In the abstracted map model, how are one-way streets or no-left-turn rules represented? | As directed edges (arcs) in the graph
In the abstracted map model, how are distances between junctions represented? | As weights on the edges or arcs of the graph
What four questions should you consider when devising an abstract model? | (1) What specific problem needs to be solved? (2) Can it be broken down into milestones? (3) What elements impact the solution? (4) If an element were removed, would it affect the solution?
When devising an abstract model, why should you ask "if an element were removed, would it impact the solution"? | To identify which elements are truly necessary; if removing an element has no effect, it is unnecessary detail that should be excluded from the model
What is machine code (first generation language)? | A programming language that uses binary (0s and 1s) to create programs, executed directly by the processor
What is assembly language (second generation language)? | A low-level language that uses mnemonics to represent groups of binary instructions, making programs easier to write than in machine code
A low-level language that uses mnemonics to represent binary instructions — what generation of language is this? | Second generation (assembly language)
What advantage does assembly language have over machine code in terms of abstraction? | It abstracts the underlying binary implementation away from the developer, so they do not need to work directly with 0s and 1s
What is a key disadvantage of assembly language? | Each processor family has its own instruction set, so assembly programs must be rewritten to run on different processor architectures
Give examples of third generation (high-level) programming languages | BASIC, FORTRAN, Python, Java, C
How do third generation languages demonstrate abstraction over assembly language? | They abstract long, complicated sequences of low-level instructions into shorter, more readable instructions (e.g. multiplication takes many assembly lines but one line in Python: A = B * C)
What do high-level languages allow developers to ignore through abstraction? | How data is stored in memory and the specifics of how instructions are carried out in the processor
How does abstraction in programming languages benefit developers? | It lets them focus on solving the problem rather than worrying about technical hardware details, similar to how a car driver does not need to understand engine mechanics to drive
Explain the relationship between programming language generations and levels of abstraction | Each successive generation provides a higher level of abstraction: machine code (1st gen) has no abstraction, assembly (2nd gen) abstracts binary into mnemonics, high-level languages (3rd gen) abstract complex instruction sequences into simple readable statements
How does programming language abstraction differ from data abstraction? | Programming language abstraction hides low-level hardware instructions behind higher-level syntax. Data abstraction hides how data is stored and represented in memory, allowing use of structures like queues without knowing their implementation
Why are low-level languages described as "more difficult to write" than high-level languages? | Machine code requires understanding specific binary codes, and assembly requires knowing processor-specific mnemonics and instruction sets, whereas high-level language syntax parallels natural language
How has the abstraction provided by high-level languages made coding more accessible? | High-level syntax resembles natural language and is considerably easier to learn, making programming accessible to non-specialists who do not need to understand hardware details
What are primitive data types, and how does abstraction relate to them? | Primitive data types (integers, strings, booleans) have their storage and binary representation hidden from the programmer, so they can be used without knowing how they are implemented in memory
Give an example of how a data structure is an abstraction of a real-world scenario | A queue data structure models a real-world queue (e.g. at a fast-food restaurant): items are added to the back and removed from the front (FIFO), with a maximum capacity
What is the underlying implementation chain of a queue data structure? | A queue is a modified array, which is a collection of variables, which are collections of bytes, which are collections of bits, which are collections of flip-flops
Why does a programmer not need to understand the underlying implementation of a queue? | Through data abstraction, they only need to know they can add to and remove from the queue; how it is stored in memory is hidden
What does the statement "abstraction separates what a program does from how it does it" mean? | The user or programmer interacts with the functionality (what it does) without needing to understand the underlying implementation details (how it achieves this)
What are multiple levels of abstraction? | Layers where each level performs a different role; highest levels are closest to the user (providing interfaces), lowest levels handle execution through machine code
In a multi-layered system, what do the highest levels of abstraction typically handle? | Providing an interface for the user to interact with the hardware
In a multi-layered system, what do the lowest levels of abstraction typically handle? | Actually performing tasks through the execution of machine code
How is abstraction used in the TCP/IP networking model? | The model separates network communication into four abstracted layers (application, transport, internet, link), each handling a different part of the process without needing to know how the other layers work
Name the four layers of the TCP/IP model from highest to lowest | Application, Transport, Internet, Link
Why are standards and protocols important in layered abstraction models like TCP/IP? | They ensure compatibility between layers so each layer can be dealt with individually, with details about other layers being hidden
How is abstraction applied in object-oriented programming (OOP)? | Objects are abstractions of real-world entities; attributes abstract an object's characteristics and methods abstract the actions it can perform
In OOP, what do attributes and methods represent in terms of abstraction? | Attributes are an abstraction for an object's characteristics; methods are an abstraction for the actions a real-world object can perform
What is the difference between abstraction and reality? | Reality is complex with many variables; abstraction is a simplified representation that uses computational structures (variables, tables, databases, objects) to model only the relevant aspects of reality
When devising an abstract model, why is it important to consider who the target audience is? | The audience's level of expertise determines how much detail should be included or removed; non-experts need more simplification to avoid confusion
When devising an abstract model, why should you consider how the model will be used? | The intended use determines the format, level of detail, and factors like convenience, affordability, and ease of access
How does representational abstraction differ from procedural abstraction? | Representational abstraction simplifies a problem by removing irrelevant details to focus on key features. Procedural abstraction hides how a subroutine achieves its result, modelling only what it does so it can be reused as a black box
