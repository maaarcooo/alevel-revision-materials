What is the Basic Input Output System (BIOS)? | A program that initialises and tests system hardware, then loads the OS from the hard disk into RAM when the computer is turned on
A program that initialises/tests hardware and loads the OS into RAM on startup -- what is this called? | Basic Input Output System (BIOS)
What is a device driver? | A program supplied with a peripheral device that allows the OS to control and communicate with the device
A program that allows the OS to control and communicate with a peripheral device -- what is this? | A device driver
What is a distributed operating system? | An OS spread over multiple computer servers on a network, acting as a single system to parallel process a job
An OS spread across multiple networked servers that acts as one system for parallel processing -- what type of OS is this? | Distributed operating system
What is an embedded operating system? | A specialised OS with limited resources and functionality, built in to control a single machine
A specialised OS with limited functionality built into a single device -- what type of OS is this? | Embedded operating system
What is the First Come First Served scheduling algorithm? | Processes are dealt with in the order they arrive, like a queue
What is intermediate code? | Code partly translated between high-level and machine language, produced by a compiler
What is an interrupt? | A signal from hardware, software, or the clock to alert the CPU. If it has higher priority than the current task, the current routine pauses and resumes after the interrupt is executed
What happens during an Interrupt Service Routine (ISR)? | If the interrupt has higher priority than the current task, register contents are temporarily transferred onto the system stack at the end of the current FDE cycle, then the interrupt is handled
Where are register contents stored during an ISR? | On the system stack
At what point in execution are register contents saved during an ISR? | At the end of the current fetch-decode-execute cycle
What is memory management? | The efficient organisation and allocation of main memory to the programs currently in use
What is the Multi-level Feedback Queues scheduling algorithm? | A scheduling algorithm using multiple queues with different priorities, where jobs can be moved between queues
How does Multi-level Feedback Queues differ from Round-Robin? | MLFQ uses multiple queues with different priorities and jobs can move between them. Round-Robin uses a single queue with equal time slices for all processes
What is a multi-tasking operating system? | An OS capable of running multiple tasks simultaneously
What is a multi-user operating system? | An OS with one mainframe and multiple terminals, allowing multiple users to access the computer's resources. Each user gets a time slice of the processor
What is an operating system? | A set of programs managing the operation of the computer, loaded into RAM every time it is turned on. It bridges the user to the hardware
A set of programs that manages computer operation and bridges the user to the hardware -- what is this? | The operating system
What is paging? | Partitioning memory into fixed-sized physical divisions called pages. Processes are assigned an appropriate number of pages
What is a real-time operating system? | An OS where data is processed as it arrives, with responses generated within a guaranteed timeframe
An OS that processes data as it comes and guarantees response times -- what type of OS is this? | Real-time operating system
What is the Round-Robin scheduling algorithm? | Each process is given an equal time slice on a first-in-first-out basis. If a process doesn't finish within its time slice, it joins the end of the queue
What is scheduling? | Allocating processor time to each application to ensure processor time is used as efficiently as possible when multitasking
Allocating processor time to applications for efficient multitasking -- what is this called? | Scheduling
What is segmentation? | Partitioning memory into variable-sized logical divisions called segments. A large program can be executed by consecutively running its segments
How does paging differ from segmentation? | Paging uses fixed-sized physical divisions (pages). Segmentation uses variable-sized logical divisions (segments)
What is the Shortest Job First scheduling algorithm? | Picks the process with the shortest estimated total running time and runs it until it finishes
What is the Shortest Remaining Time scheduling algorithm? | Picks the process with the shortest estimated time remaining. If a new process with shorter remaining time is added, the scheduler switches to it
How does Shortest Remaining Time differ from Shortest Job First? | SJF runs the shortest total job to completion (non-preemptive). SRT can switch to a new process if it has shorter remaining time (preemptive)
What is a virtual machine? | Any instance where software is used to take on the function of a machine, including running intermediate code or an OS within another OS
What is virtual memory? | An allocated area of secondary storage where pages of inactive jobs are swapped into, to free up enough RAM for the current job
Why is virtual memory needed? | When RAM is full, inactive pages are swapped to secondary storage to free up RAM for the current job
What is an application? | A program that can be run on a computer, allowing the user to carry out specific tasks
What is an assembler? | A translator that converts assembly language into machine code
A translator that converts assembly language into machine code -- what is this called? | An assembler
What is closed source software? | Proprietary software sold with a licence and restrictions on usage. The source code is not available to users
What is code generation? | The third and final stage of compilation, where an equivalent machine code program is produced
In which stage of compilation is the machine code program produced? | Code generation (the third and final stage)
What is compilation? | The process of analysing high-level language source code and converting it into machine code
What is a compiler? | A translator that converts high-level language to machine code
What is an interpreter? | A translator that checks a source program for syntax errors line by line, translates it to machine code, and executes each line
How does a compiler differ from an interpreter? | A compiler translates the entire source code into machine code before execution. An interpreter translates and executes one line at a time, checking for syntax errors as it goes
What is lexical analysis? | The first stage of compilation: extra spaces and comments are removed, the code is searched for simple errors, keywords/constants/variables are replaced by tokens, and variable names are loaded into the symbol table
What are the four actions performed during lexical analysis? | 1) Remove spaces and comments, 2) Search for simple errors, 3) Replace keywords/constants/variables with tokens, 4) Load variable names into the symbol table
In which stage of compilation are tokens created? | Lexical analysis (the first stage)
What is a library (in programming)? | A collection of pre-compiled programs that can be loaded into a program and run whenever required
A collection of pre-compiled programs that can be loaded and run in other programs -- what is this? | A library
What is a linker? | A program that places the appropriate machine addresses in the call and return instructions of a compiled program, linking all required object code files and modules together
What is a loader? | A program that loads the executable object program and its associated libraries into memory before it is run
How does a linker differ from a loader? | A linker connects separate object code files and modules by inserting correct addresses. A loader places the linked executable and its libraries into memory for execution
What is open source software? | Software whose source code is freely available to view, redistribute, or modify
How does open source differ from closed source software? | Open source: source code is freely available to view, redistribute, or modify. Closed source: source code is unavailable, sold with a licence and usage restrictions
What is optimisation (in compilation)? | During code generation, removing redundancies from the object code to make it as efficient as possible while producing the same result
What is syntax analysis? | The second stage of compilation: statements, expressions, and tokens are checked for syntax errors using syntax diagrams
In which stage of compilation are syntax diagrams used to check for errors? | Syntax analysis (the second stage)
What are the three stages of compilation in order? | 1) Lexical analysis, 2) Syntax analysis, 3) Code generation
What is a translator? | A program that converts code from one computer language to another
A program that converts code from one language to another -- what is this called? | A translator
What are utilities? | System software with a specific maintenance purpose, such as optimising performance, diagnosing issues, backing up files, or setting up firewalls
What are agile methodologies? | An iterative process producing incremental prototypes over short, fast-paced sprints. Each prototype is user-tested and feedback is incorporated into future sprints
An iterative development process using short sprints, user testing, and incremental prototypes -- what methodology is this? | Agile methodologies
What is extreme programming? | A type of agile methodology that is more responsive to changing user needs, with short development cycles, very frequent releases, and checkpoints for changing or adding requirements
How does extreme programming differ from standard agile? | Extreme programming has shorter development cycles, more frequent releases, and explicit checkpoints for changing or adding user requirements
What is Rapid Application Development (RAD)? | A method using repeated prototyping, continual evaluations, and strict time limits to respond quickly to changes. The user quickly receives a reduced-functionality mock-up
What is the Spiral Model? | An iterative version of the Waterfall model where stages are refined and repeated. The first cycle produces an initial prototype, and each successive cycle produces a refined prototype
How does the Spiral Model differ from the Waterfall model? | Waterfall completes stages linearly, one at a time. The Spiral Model repeats and refines stages iteratively, producing progressively better prototypes each cycle
What is the Waterfall Lifecycle? | Each stage of development is completed one at a time in linear order. Results from one stage feed into the next. Previous stages can be revisited, but subsequent stages must be reworked
What is assembly language? | A low-level language closely related to machine code that uses descriptive names and mnemonics for instructions, and is related to the design of the computer
A low-level language using mnemonics instead of binary -- what is this? | Assembly language
What are attributes (in OOP)? | Data recorded as a variable associated with an object
What is a class (in OOP)? | A template defining the attributes and methods that can be used to create objects
A template defining attributes and methods used to create objects -- what OOP concept is this? | A class
What is direct addressing? | The simplest addressing mode where the operand stores the memory address of the value to be operated on
What is encapsulation? | Maintaining data integrity by only allowing class methods to access the data in an object's attributes
Only allowing class methods to access an object's attribute data -- what OOP principle is this? | Encapsulation
What is immediate addressing? | An addressing mode where the operand itself is the value to be operated on (the data in the address field is constant)
How does immediate addressing differ from direct addressing? | In immediate addressing, the operand IS the value. In direct addressing, the operand is the memory address where the value is stored
What is indexed addressing? | An addressing mode where the operand address is calculated by adding a constant from the index register to the absolute address in the instruction
What is indirect addressing? | An addressing mode where the operand stores the address of the location that holds the memory address of the required data
How does indirect addressing differ from direct addressing? | In direct addressing, the operand holds the address of the data. In indirect addressing, the operand holds the address of a location that itself contains the address of the data
What is inheritance (in OOP)? | Subclasses inheriting the methods and attributes of their parent class (super class)
Subclasses receiving the methods and attributes of their parent class -- what OOP concept is this? | Inheritance
What is a method (in OOP)? | A program subroutine that represents an action an object can perform
What is an object (in OOP)? | An instance of a class whose behaviour depends on how its attributes were defined
An instance of a class with its own attributes and behaviour -- what OOP concept is this? | An object
What is polymorphism? | Objects of different classes can use the same method to perform an action
Different classes using the same method name to perform an action -- what OOP concept is this? | Polymorphism
What is a procedural language? | A high-level language where statements are grouped into self-contained blocks known as functions and procedures, with built-in data types and data structures
What is a programming paradigm? | A style of computation and programming, chosen according to the problem at hand
How do object-oriented languages differ from procedural languages? | OOP organises code into objects (instances of classes) with their own attributes and behaviours. Procedural languages organise code into functions and procedures that execute sequentially
Name the four addressing modes in assembly language | Direct, immediate, indexed, and indirect addressing
What are object-oriented languages? | Languages where code is made of objects (instances of classes) that have their own attributes and behaviours and can interact with each other
