What is asymmetric encryption? | An encryption technique where a public key (available to everyone) encrypts the data, and the paired private key (known only by the recipient) decrypts it
An encryption method using a public key to encrypt and a separate private key to decrypt — what is this called? | Asymmetric encryption
What is dictionary coding? | A type of lossless compression where text entries matching a dictionary are substituted with unique shorter codes that can later be translated back
A lossless compression method that replaces text entries with shorter codes using a lookup table — what is this called? | Dictionary coding
What is hashing? | A one-way transformation of data into an abbreviated form called a hash value, used to validate credentials like passwords without revealing the original data
A one-way transformation that converts data into an abbreviated value for validating credentials — what is this process called? | Hashing
Why is hashing described as "one-way"? | The original data cannot be recovered from the hash value — the transformation is irreversible
What is lossless compression? | A compression algorithm that retains all original data by storing only the instructions needed to reconstruct the original file
A compression method where the original file can be perfectly reconstructed with no data loss — what type of compression is this? | Lossless compression
What is lossy compression? | A compression algorithm that removes non-essential data from a file, causing a noticeable decrease in accuracy. The lost data is non-recoverable
A compression method that permanently removes non-essential data to reduce file size — what type of compression is this? | Lossy compression
How does lossy compression differ from lossless compression? | Lossy permanently removes non-essential data (irreversible, smaller files, reduced accuracy). Lossless retains all data by storing reconstruction instructions (reversible, larger files, perfect accuracy)
What is run-length encoding (RLE)? | A type of lossless compression where repeated occurrences of the same data value are stored as a single value with its count
A lossless compression technique that stores repeated consecutive data values as a single value and a count — what is this called? | Run-length encoding (RLE)
What is symmetric encryption? | An encryption technique where the same key is used to both encrypt and decrypt data. A copy of the key must be transferred between both parties
An encryption method where the same single key is used for both encryption and decryption — what is this called? | Symmetric encryption
How does symmetric encryption differ from asymmetric encryption? | Symmetric uses one shared key for both encryption and decryption (key must be transferred). Asymmetric uses two keys: a public key to encrypt and a private key to decrypt (no key transfer needed)
What is a flat file database? | A database where a single table structure is used to store all the data
A database that stores all data in a single table — what type of database is this? | Flat file database
What is a relational database? | A database where separate tables are made for each entity, and relationships between entities are represented by foreign keys
A database using separate tables for each entity with foreign keys representing relationships — what type of database is this? | Relational database
How does a relational database differ from a flat file database? | A flat file stores all data in one table (leading to redundancy). A relational database splits data into separate entity tables linked by foreign keys (reducing redundancy)
What is a primary key? | A unique identifier that identifies each record in a table
A unique attribute that identifies each record in a database table — what is this called? | Primary key
What is a foreign key? | A linking attribute that joins two tables in a relational database by being a primary key in one table and a foreign key in the other
An attribute that links two tables by referencing a primary key from another table — what is this called? | Foreign key
What is a secondary key? | A key used as an alternative index to access or sort records more quickly, but less accurately, than the primary key
A key that provides an alternative, faster (but less precise) way to access or sort records — what is this called? | Secondary key
What is indexing in the context of databases? | The process of creating an index of primary keys so that the location of any record can be quickly retrieved given its primary key
What is normalisation? | The formal process of optimally designing data tables by reducing data redundancy and repetition, converting them into normal forms
The formal process of reducing redundancy in database tables by converting them into normal forms — what is this called? | Normalisation
What is First Normal Form (1NF)? | A table with no repeating attributes, where the intersection of each record and attribute produces exactly one value
What is Second Normal Form (2NF)? | A table in 1NF where data that repeats across multiple records is removed into a new table with appropriate relationships (no partial dependencies)
What is Third Normal Form (3NF)? | A table in 2NF where all non-key attributes are fully dependent on the primary key (no non-key dependencies)
What must a table satisfy before it can be in 2NF? | It must first be in 1NF (no repeating attributes, each cell contains exactly one value)
What must a table satisfy before it can be in 3NF? | It must first be in 2NF (which itself requires 1NF, plus no partial dependencies)
What is redundancy in a database? | The unnecessary repetition of a field across multiple tables. However, databases should have redundancy as identical backup copies for recovery purposes
What is referential integrity? | Keeping a database consistent by ensuring that any changes to data or relationships in one table are properly accounted for in all linked tables
The principle that changes to data in one table must be reflected in all linked tables to maintain consistency — what is this called? | Referential integrity
What is entity-relationship modelling? | A method of abstractly describing data tables and the relationships between them visually, used to reduce redundancy and help construct a relational database
What is record locking? | A technique that prevents simultaneous access to data by locking a record while it is being edited or updated, preventing inconsistencies
A technique that locks a database record during editing to prevent simultaneous access and inconsistencies — what is this called? | Record locking
Why is record locking needed in databases? | Without it, two users could edit the same record simultaneously, causing one update to overwrite the other and creating data inconsistencies
What is transaction processing? | Ensuring that any logical operation or change in a database (transaction) conforms to ACID rules for reliable processing
What does ACID stand for in transaction processing? | Atomicity, Consistency, Isolation, Durability
What is atomicity (in ACID)? | All transactions should either fully succeed or fully fail — they must never be only partially processed
What is consistency (in ACID)? | Each transaction must obey the defined validation rules of the database to maintain referential integrity
What is isolation (in ACID)? | No transaction should overwrite other simultaneously occurring transactions. The same results must be obtained whether transactions are processed concurrently or sequentially
What is durability (in ACID)? | Once a transaction has begun, it must be completed under all circumstances
What is circuit switching? | A method for sending data between two nodes by first creating a dedicated communication channel. All data follows this same path for the entire duration of communication
A network data transfer method where a dedicated channel is established first and all data follows the same path — what is this called? | Circuit switching
What is packet switching? | A method for sending data over a network by breaking data into packets that are sent independently via potentially different routes, then reassembled at the destination
A network data transfer method where data is split into independently routed packets and reassembled at the destination — what is this called? | Packet switching
How does circuit switching differ from packet switching? | Circuit switching creates one dedicated path used for the entire communication. Packet switching breaks data into packets sent independently via potentially different routes, then reassembles them at the destination
What is a client-server network? | A network where client computers connect to one or more powerful central servers that handle service requests and provide resources
A network where multiple computers connect to a central server that manages requests and resources — what type of network is this? | Client-server network
What is a peer-to-peer network? | A network where computers are connected to each other directly and share resources and workloads without any central server
A network where all computers share resources equally with no central server — what type of network is this? | Peer-to-peer network
How does a client-server network differ from a peer-to-peer network? | Client-server has a central server managing resources and requests. Peer-to-peer has no central server — all computers share resources and workloads equally
What is a Local Area Network (LAN)? | A network connecting computing devices within a single geographical area or site
What is a Wide Area Network (WAN)? | A network connecting computing devices over a large geographical area, typically using third-party carriers
How does a LAN differ from a WAN? | A LAN covers a single geographical site (e.g. a school). A WAN spans large geographical areas and typically relies on third-party communication carriers
What is the Domain Name System (DNS)? | A hierarchical naming system that maps human-readable domain names to IP addresses that locate and identify resources on a network
A hierarchical system that translates human-readable domain names into IP addresses — what is this called? | Domain Name System (DNS)
What is encryption? | The process of converting original data (plaintext) into a form unreadable by unauthorised users (ciphertext), using an encryption algorithm (cipher)
What is a firewall? | A security application that monitors incoming and outgoing network traffic, designed to prevent external users from gaining unauthorised access between two networks
A security application that monitors network traffic and prevents unauthorised external access — what is this called? | A firewall
What is a proxy server? | A server application that intercepts all packets entering and leaving a network to hide the true network addresses from recipients, restrict user access to certain data, and isolate the network from external networks
A server that hides internal network addresses, restricts access, and isolates the network from external networks — what is this called? | A proxy server
What is protocol layering? | A technique that simplifies network design by dividing a complex system into functional layers, each with assigned protocols to perform tasks and communicate with adjacent layers
Why is protocol layering used in network design? | It simplifies complex network design by breaking it into separate functional layers, each responsible for specific tasks and communicating only with adjacent layers
What is the TCP/IP stack? | A suite of networking protocols consisting of 4 connected layers that allow networked computers to communicate. Data packets pass through these layers when sent or received
The [...]-layer protocol suite that governs how networked computers communicate by passing data packets through each layer | 4 (the TCP/IP stack has 4 layers)
What is CSS (Cascading Style Sheets)? | A style language used to define the layout, design, and presentation of a webpage written in a markup language
A style language that controls the layout, design, and presentation of webpages — what is it called? | CSS (Cascading Style Sheets)
What is HTML (HyperText Markup Language)? | A markup language used to write the contents of webpages on the World Wide Web
A markup language used to define the content and structure of webpages — what is it called? | HTML (HyperText Markup Language)
How does HTML differ from CSS in web development? | HTML defines the content and structure of a webpage. CSS defines the layout, design, and visual presentation of the content
What is JavaScript in the context of web development? | An interpreted, object-oriented programming language used to create interactive elements and add functionality to dynamic web pages within browsers
An interpreted programming language used to add interactivity and dynamic functionality to web pages — what is it? | JavaScript
What is client-side processing? | Data processing and operations performed by the client's browser using the user's local computer resources. Typically used for less critical code
What is server-side processing? | Data processing and operations performed by the web server. Data is sent from the client's browser to the server for secure processing, and the output is returned to the client
How does client-side processing differ from server-side processing? | Client-side runs in the user's browser using local resources (for less critical tasks). Server-side runs on the web server (for secure processing), with data sent to and results returned from the server
Why is server-side processing preferred for sensitive operations? | Processing occurs on the web server rather than the user's browser, keeping sensitive data and logic secure and hidden from the client
What is the PageRank algorithm? | An algorithm developed by Google that ranks webpages in search results by their quality and importance, measured by counting the number of links pointing to that page
An algorithm that ranks webpages by counting the number of inbound links to assess quality and importance — what is this called? | PageRank algorithm
What is search engine indexing? | The process of collecting, parsing, and storing data from websites to enable fast and accurate information retrieval by the search engine using an index of webpages
The process of collecting and storing website data to enable fast retrieval by a search engine — what is this called? | Search engine indexing
