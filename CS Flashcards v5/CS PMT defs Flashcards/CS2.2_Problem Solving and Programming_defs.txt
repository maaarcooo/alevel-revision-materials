What is branching? | A programming control structure where the code selects one or more alternative paths depending on the evaluation of a boolean expression
A control structure that selects alternative paths based on a boolean expression — what is this called? | Branching
What is a function in programming? | A subroutine that performs a task or calculation and always returns a value. It can be called in an expression or assigned to a variable
A subroutine that always returns a value and can be used in an expression — what is this called? | A function
What is a global variable? | A variable declared in the main program, outside any subroutine, that can be accessed anywhere in the program
A variable declared outside all subroutines that is accessible from anywhere in the program — what term describes this? | Global variable
What is an Integrated Development Environment (IDE)? | A software package for developing and implementing code, with features for editing, debugging, version control, testing and compilation
A software package with features for editing, debugging, version control, testing and compilation — what is this? | An Integrated Development Environment (IDE)
What is iteration? | A programming control structure where a set of statements is repeated either a fixed number of times or until a condition is met
A control structure that repeats a set of statements a fixed number of times or until a condition is met — what is this called? | Iteration
What is a local variable? | A variable declared within a subroutine that only exists and can be used within that subroutine
A variable that only exists within the subroutine where it was declared — what term describes this? | Local variable
What is modularity? | Breaking down a complex problem into simpler, self-contained components called modules, where each module implements a specific subtask
Breaking a complex problem into simpler, self-contained modules that each handle a specific subtask — what technique is this? | Modularity
What is Object Oriented Programming (OOP)? | A programming paradigm where the system is viewed as a set of objects, each with attributes (data) and methods (procedures), that interact with each other. All processing is done by objects
A paradigm where a system is modelled as interacting objects, each with their own attributes and methods — what is this? | Object Oriented Programming (OOP)
What is parameter passing by reference? | Passing the address or pointer of the required value into a subroutine, so the subroutine operates on the original data
Passing the address or pointer of a value into a subroutine rather than a copy — what type of parameter passing is this? | Parameter passing by reference
What is parameter passing by value? | Creating a temporary local copy of the actual value of a variable and passing that copy into the subroutine
Creating a temporary local copy of a variable's value to pass into a subroutine — what type of parameter passing is this? | Parameter passing by value
How does parameter passing by reference differ from passing by value? | By reference passes the address of the original variable, so changes inside the subroutine affect the original. By value passes a copy, so changes inside the subroutine do not affect the original
What are parameters? | The data values or structures required to be passed into a subroutine
What is a procedure? | A subroutine called by writing its name in the code that does not have to return a value
A subroutine that is called by name and does not have to return a value — what is this called? | A procedure
How does a function differ from a procedure? | A function always returns a value and can be used in expressions. A procedure does not have to return a value and is called simply by writing its name
What is recursion? | A programming technique where a subroutine calls itself repeatedly until a base case is met
A subroutine that calls itself until a base case is reached — what technique is this? | Recursion
Why must a recursive subroutine have a base case? | To ensure the recursion terminates and does not result in an infinite loop (or stack overflow)
What is a sequence? | A programming control structure where statements are executed one after another in the order they appear in the script
A control structure where statements execute one after another in written order — what is this called? | Sequence
What is a subroutine? | A uniquely named section of code written to perform a specific task within a program
A uniquely named block of code that performs a specific task within a program — what general term describes this? | Subroutine
Name the three main programming control structures | Sequence, branching (selection), and iteration
How does a global variable differ from a local variable? | A global variable is declared in the main program and accessible everywhere. A local variable is declared inside a subroutine and only accessible within that subroutine
What is backtracking? | An algorithm that incrementally builds a solution by trying different sequences and abandoning a path as soon as it determines it cannot lead to a valid solution
An algorithm that tries paths incrementally and abandons a path when it cannot lead to a valid solution — what is this called? | Backtracking
What is a computable problem? | A problem for which every instance can be solved in a finite number of steps by means of an algorithm
A problem where every instance can be solved in a finite number of steps using an algorithm — what term describes this? | A computable problem
What are computational methods? | Methods of solving problems by devising and implementing algorithms using some form of computation
What is data mining? | An algorithm that analyses large data sets to uncover trends and relationships between variables
Analysing large data sets to discover trends and relationships between variables — what technique is this? | Data mining
What is divide and conquer? | An algorithm design technique that decomposes and solves problems by reducing the problem size with each iteration until the sub-problem becomes directly solvable
An algorithm design technique that repeatedly reduces the problem size until sub-problems become directly solvable — what is this called? | Divide and conquer
What are heuristics? | 'Rule of thumb' algorithms that produce a valid but sub-optimal (approximate) solution for hard (intractable) problems
An approach that produces a valid but sub-optimal solution for intractable problems — what is this called? | A heuristic
Why are heuristics used instead of exact algorithms for some problems? | Some problems are intractable (no known efficient exact solution), so a heuristic provides a good-enough approximate answer in a reasonable time
What is performance modelling? | Simulating the behaviour of a system under different virtual user and system loads using mathematical approximation
Simulating system behaviour under different loads using mathematical approximation — what is this called? | Performance modelling
What is pipelining? | Splitting a task into parts and identifying subtasks that can be processed simultaneously, overlapping the processing of each part
Splitting a task into parts so that subtasks can be processed simultaneously — what technique is this? | Pipelining
What is problem decomposition? | Splitting a given problem into smaller, solvable sub-problems that are easier to understand, maintain and program
Splitting a problem into smaller, more manageable sub-problems — what process is this? | Problem decomposition
What is problem recognition? | The ability to identify the most effective strategy to solve a given problem
The ability to identify the most effective strategy for solving a problem — what is this called? | Problem recognition
What is visualisation in computational methods? | Using a visual representation of an algorithm or data structure to present a problem and its solution in a more human-readable form
Using visual representations to make algorithms or data structures easier for humans to understand — what computational technique is this? | Visualisation
How does divide and conquer differ from problem decomposition? | Problem decomposition splits a problem into smaller sub-problems of potentially different types. Divide and conquer specifically reduces the same problem into smaller instances of itself with each iteration until directly solvable
How does backtracking differ from a brute force approach? | Backtracking abandons a path as soon as it determines it cannot lead to a valid solution (pruning), making it more efficient. Brute force exhaustively tries all possible solutions without pruning
