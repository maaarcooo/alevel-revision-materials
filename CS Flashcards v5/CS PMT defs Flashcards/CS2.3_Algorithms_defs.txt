What is the A* algorithm? | A generalisation of Dijkstra's algorithm that approximately finds the shortest path between two nodes by combining a cost function (distance from source to current node) and a heuristic function (estimated distance from next node to destination)
A pathfinding algorithm that combines actual distance travelled with an estimated distance to the destination to approximately find the shortest path — what is this? | A* algorithm
What is Big O notation? | A way of classifying algorithms by setting an upper bound for their worst-case time or space complexity, expressed as a mathematical function of input size
A classification system that expresses the worst-case time or space complexity of an algorithm as a mathematical function of input size — what is this called? | Big O notation
What is binary search? | An O(log n) search algorithm that finds an item in a sorted list by repeatedly halving the sublist that could contain the item
An O(log n) algorithm that searches a sorted list by repeatedly halving the search space — what is this? | Binary search
What is breadth-first traversal? | A graph traversal method that uses a queue to visit all neighbours of the current node before moving on to their neighbours, continuing until the entire graph is explored
A graph traversal that uses a queue and visits all neighbours of a node before progressing deeper — what is this? | Breadth-first traversal
What is bubble sort? | An O(n^2) sorting algorithm that iterates through a list, comparing each element to its successor and swapping them if they are in the wrong order, repeating until no swaps are needed
An O(n^2) sort that repeatedly compares adjacent elements and swaps them if out of order, stopping when a full pass requires no swaps — what is this? | Bubble sort
What is depth-first traversal? | A graph traversal method that uses a stack to travel as far as possible along one route before backtracking and exploring remaining routes until the entire graph is visited
A graph traversal that uses a stack to go as deep as possible along a path before backtracking — what is this? | Depth-first traversal
What is Dijkstra's shortest path algorithm? | An algorithm that finds the shortest path between two nodes on a graph by using a priority queue to track the shortest known distance (cost) to each node from the starting node until the destination is reached
An algorithm that uses a priority queue to track cumulative distances from a start node, finding the shortest path to a destination — what is this? | Dijkstra's shortest path algorithm
What is insertion sort? | An O(n^2) sorting algorithm that divides a list into sorted and unsorted sections, taking elements from the unsorted section one by one and placing them in the correct position in the sorted section
An O(n^2) sort that maintains a sorted section and inserts unsorted elements into their correct position one at a time — what is this? | Insertion sort
What is linear search? | An O(n) search algorithm that checks each element in a list sequentially until the target item is found or the end of the list is reached
An O(n) algorithm that searches a list by checking every element in sequence — what is this? | Linear search
What is merge sort? | An O(n log n) divide-and-conquer sorting algorithm that recursively halves a list into sublists of length 1, then merges them back together in sorted order
An O(n log n) divide-and-conquer sort that splits a list into single-element sublists and then merges them back in order — what is this? | Merge sort
What is quick sort? | An O(n log n) sorting algorithm that selects a pivot element when splitting the list — elements greater than the pivot go into one sublist and the rest into another — repeating recursively until all sublists are length 1
An O(n log n) sort that partitions elements around a chosen pivot, placing larger elements in one sublist and smaller in another, recursively — what is this? | Quick sort
What does O(1) mean? | The algorithm takes a constant amount of time or memory regardless of input size
An algorithm whose execution time or memory usage does not change with input size has what complexity? | O(1) — constant time/space
What does O(n) mean? | The algorithm's execution time or memory usage is directly proportional to the input size
An algorithm whose time or space grows in direct proportion to input size has what complexity? | O(n) — linear time/space
What does O(log n) mean? | The algorithm's execution time or memory usage grows logarithmically with input size (lower complexity than polynomial)
An algorithm whose time or space grows logarithmically with input size has what complexity? | O(log n) — logarithmic time/space
What does O(n^k) mean? | The algorithm's execution time or memory usage is proportional to the input size raised to a constant power k (e.g. O(n^2), O(n^3))
An algorithm whose time or space is proportional to input size raised to a constant power has what complexity? | O(n^k) — polynomial time/space
What does O(2^n) mean? | The algorithm's execution time or memory usage grows exponentially with input size (higher complexity than polynomial)
An algorithm whose time or space grows exponentially, faster than any polynomial, has what complexity? | O(2^n) — exponential time/space
What is time complexity? | A measure of how much time an algorithm needs to solve a problem as a function of input size
What is space complexity? | A measure of how much memory an algorithm needs to solve a problem as a function of input size
A measure of the memory required by an algorithm for a given input size — what is this? | Space complexity
A measure of the time required by an algorithm for a given input size — what is this? | Time complexity

What time complexity does linear search have? | O(n)
What time complexity does bubble sort have? | O(n^2)
What time complexity does insertion sort have? | O(n^2)
What time complexity does merge sort have? | O(n log n)
What time complexity does quick sort have? | O(n log n)
What data structure does breadth-first traversal use? | A queue
What data structure does depth-first traversal use? | A stack
What data structure does Dijkstra's algorithm use to track shortest distances? | A priority queue
What precondition must be met before binary search can be used? | The list must be sorted
How does A* differ from Dijkstra's algorithm? | A* adds a heuristic function that estimates the remaining distance to the destination, allowing it to approximate the shortest path more efficiently. Dijkstra's explores purely based on cumulative cost from the source without estimating remaining distance.
What are the two functions that A* combines? | The cost function (actual distance from source to current node) and the heuristic function (estimated distance from current node to destination)
How does breadth-first traversal differ from depth-first traversal? | Breadth-first uses a queue and explores all neighbours at the current depth before going deeper. Depth-first uses a stack and explores as far as possible along one path before backtracking.
How does merge sort differ from quick sort? | Merge sort splits the list in half each time and merges sublists in sorted order. Quick sort selects a pivot and partitions elements around it. Both are O(n log n), but merge sort always divides evenly while quick sort's efficiency depends on pivot choice.
How does binary search differ from linear search? | Binary search requires a sorted list and halves the search space each step at O(log n). Linear search works on any list but checks elements one by one at O(n).
Why is binary search more efficient than linear search for large sorted datasets? | Binary search eliminates half the remaining elements with each comparison, giving O(log n) complexity, whereas linear search may need to check every element giving O(n)
Why is bubble sort considered inefficient for large lists? | It has O(n^2) time complexity, meaning the number of comparisons grows quadratically with input size, making it very slow for large datasets
Rank these complexities from fastest to slowest growth: O(n), O(1), O(2^n), O(n^2), O(log n), O(n log n) | O(1), O(log n), O(n), O(n log n), O(n^2), O(2^n)
Does Big O notation describe best-case or worst-case performance? | Worst-case performance — it sets an upper bound on time or space complexity
Explain why Dijkstra's algorithm uses a priority queue | A priority queue always processes the unvisited node with the smallest cumulative distance next, ensuring that when a node is visited, the shortest path to it has been found
How does bubble sort know when the list is fully sorted? | When a complete pass through the list produces no swaps, the algorithm terminates
In insertion sort, what happens to each element taken from the unsorted section? | It is compared with elements in the sorted section and placed in its correct position
Why is merge sort described as a divide-and-conquer algorithm? | It divides the list into progressively smaller sublists (divide), then combines them back together in sorted order (conquer)
