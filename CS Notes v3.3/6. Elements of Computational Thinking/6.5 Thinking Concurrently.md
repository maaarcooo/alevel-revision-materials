# 6.5 Thinking Concurrently

## Concurrent Thinking

**Concurrent thinking** is a problem-solving mindset that allows you to identify patterns and parts of problems where **concurrency** can be applied. It is built upon the concept of concurrent processing.

The core idea is recognising where multiple tasks can progress within the same timeframe, rather than completing one task entirely before starting another.

**Example**: Recording car number plates and colours as cars pass by.
- Inefficient approach: Record all number plates first, then go back and record all colours
- Concurrent approach: Record the number plate and colour of each car before moving to the next car

### Identifying Concurrent Opportunities

To determine which parts of a problem can be tackled concurrently:
1. Assess which parts of the problem are **related**
2. Related sections can often be solved simultaneously
3. Consider **dependencies** — tasks that rely on other tasks to start or complete

**Dependency examples**:
- A roof cannot be installed until walls are built; walls cannot be built until foundations are in place
- Tax calculations cannot be completed until all income and deductibles are entered

Tasks without dependencies on each other can be processed concurrently or in parallel.

## Concurrent Processing vs Parallel Processing

| Aspect | Concurrent Processing | Parallel Processing |
|--------|----------------------|---------------------|
| **Processors** | Single processor | Multiple processors/cores |
| **Execution** | Sequential with time slicing | Truly simultaneous |
| **Appearance** | Appears simultaneous | Actually simultaneous |
| **Mechanism** | Each task given a **time slice** of processor time | Tasks run on separate processors at the same time |

**Concurrent processing** allows multiple processes to run on a single processor by giving each process a fraction of time and control before swapping to another process. Human perception is slow enough that the rapid switching creates an **illusion of simultaneous execution**.

**Parallel processing** uses multiple processor cores to execute multiple tasks genuinely at the same time.

## Concurrent Thinking vs Concurrent Processing

| Concurrent Thinking | Concurrent Processing |
|--------------------|----------------------|
| Uses the **brain** (human reasoning) | Uses a **computer processor** |
| A mindset for identifying concurrency opportunities | The actual execution of concurrent tasks |

## Benefits of Concurrent Processing

- **Increased throughput**: More tasks completed in a given timeframe (e.g., ten programs could be half-finished rather than two completed)
- **Reduced idle time**: Less time wasted waiting for user input or I/O operations, as the processor can switch to other tasks while waiting
- **Better responsiveness**: Operating system can use interrupts to swap back to waiting tasks when input arrives

## Drawbacks of Concurrent Processing

- **Slower individual completion**: With large numbers of users or computationally intensive tasks, individual processes take longer since each only receives a fraction of processor time
- **Context switching overhead**: Coordinating and switching between processes consumes resources and reduces overall program throughput
- **Not universally applicable**: Some tasks are inherently sequential and cannot be broken up or performed concurrently

## Benefits of Parallel Processing

- **Speed improvement**: Repetitive calculations on large data sets (image/video editing) complete faster when split across multiple processors
- **GPU rendering**: Graphics processing units render 3D scenes quickly by distributing computations across multiple cores
- **True multitasking**: Multiple programs or browser tabs can run on different processors simultaneously

## Drawbacks of Parallel Processing

- **Communication overhead**: Different processors running simultaneously may need to communicate, causing delays
- **Optimisation issues**: Some tasks run faster when optimised for a single processor rather than distributed across multiple processors

## Practical Applications

**Batch processing**: When running analysis on independent data sets (e.g., climate data, departmental finances), unrelated data can be processed in parallel on multiple cores for improved efficiency.

**Multiple batch programs**: Different batch programs can be interchanged and pipelined using concurrent processing on the same processor, allowing multiple programs to progress within a shorter timeframe.

**Scheduling algorithms**: Operating systems can prioritise certain concurrent processes over others, meaning larger or more important processes can finish sooner.

## Exam Tips

When explaining concurrent processing in exam questions:
1. **State the example** clearly
2. **Explain how it acts concurrently** (what tasks are being interleaved)
3. **Justify why concurrency is necessary** (what benefit it provides in that context)

**Example answer structure** (flight simulator scenario):
- Definition: Multiple processes allocated time slices and pipelined to make progress in the same timeframe
- Application: The player's plane, other planes, and weather conditions can operate independently
- Justification: Allows multiple objects and events to occur "simultaneously" and react to different events
