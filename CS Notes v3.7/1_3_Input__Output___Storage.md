# 1.1.3 Input, Output and Storage

## Input Devices

An **input device** is a hardware component that allows users to put data or commands into a computer system, which the computer then processes to produce output.

| Device | Description |
|---|---|
| **Keyboard** | The most common input device. Allows users to input text and commands by pressing keys. |
| **Mouse** | Allows users to navigate the screen, click on items, and perform other functions. |
| **Touchscreen** | Found on smartphones, tablets, and some computers. Allows interaction by touching the screen directly. Also acts as an output device (displays visual information like a monitor), so it is classified as **both input and output**. |
| **Microphone** | Captures audio input for voice commands, recording audio, or video conferencing. |
| **Webcam** | A camera used to capture video input for photos, video recording, or conferencing. |
| **Scanner** | Digitises physical documents or images, converting them into a format the computer can process. |
| **Barcode Reader** | Scans barcodes, typically used in supermarket checkouts, retail, inventory management, and access control systems. |
| **Magnetic Stripe Reader** | Reads data from magnetic stripes on cards such as credit and debit cards. |
| **Game Controller** | Used primarily for video games, allowing users to control characters and interact with the gaming environment. |
| **Joystick** | Often used for computer games, especially flight simulators. Allows more fluid movement control than a keyboard or mouse. |
| **Graphics Tablet** | Allows artists and designers to draw or sketch directly onto a computer. Useful for graphic design, 3D modelling, and other visual creative tasks. |
| **Biometric Devices** | Devices such as fingerprint scanners or facial recognition systems, used for security purposes to verify a user's identity. |

## Output Devices

An **output device** is a hardware component that receives information from a computer system and presents it to the user in a comprehensible form.

| Device | Description |
|---|---|
| **Monitor** | The most common output device. Displays visual output including text, images, and videos. |
| **Printer** | Produces a hard copy of digital documents or images on paper. Types include inkjet, laser, and 3D printers. |
| **Speakers** | Output audio from the computer such as music, sound effects, or voice. |
| **Headphones** | Similar to speakers but output audio directly to the user, providing a more personal and potentially immersive experience. |
| **Projector** | Projects video content from a computer onto a large screen or wall, useful for presentations or movie viewing. |
| **Braille Display** | A specialised device that outputs information in Braille, allowing visually impaired users to read text from the computer. |
| **Plotter** | Used for printing large, high-quality diagrams and designs, often used in engineering or architecture. |
| **VR Headset** | Provides immersive visual and audio output, primarily used for gaming and virtual simulations. |
| **Computer-Controlled Machinery** | In manufacturing or robotics, computers output commands directly to machinery to control their operation. |

## Choosing the Right Device

When recommending an input or output device for a specific situation, consider:

**User needs** dictate the type of device required. A graphic designer needs a graphics tablet, while a data entry clerk needs a keyboard with a number pad. The device must be relevant to the task.

**User skills** affect which device is appropriate. A touchscreen may be more intuitive for some users, while others prefer a mouse and keyboard.

**Environment** matters for device selection. A wireless mouse suits a clutter-free office, while a wired mouse may be better for a public computer lab to prevent theft.

**Speed and accuracy of input** vary between devices. Different devices transfer data at different rates or record information in different levels of detail.

**Cost** is always a factor. Higher-end devices offer more features but are more expensive. The budget must be weighed against whether the extra features are justified.

## Secondary Storage Devices

There are three main types of secondary storage: **magnetic**, **optical**, and **flash**. Each is suited to different types and quantities of information.

### Optical Storage

**Optical devices** are read from and written to using **lasers**. Binary information is represented by portions of the disc that either **reflect** or **scatter** the incident laser light. Data is written in **spiral tracks** on the disc's surface. Recessed portions called **pits** scatter light and represent a 0, while flat portions called **lands** reflect light and represent a 1.

| Device | Description |
|---|---|
| **CD** (Compact Disc) | Stores small quantities of information (typically **700 MB**). Most commonly used for audio files but can also store text and digital images. Small, thin, and lightweight making them very portable. Disadvantages include being easily damaged by scratches, limited storage capacity, and relatively slow transfer speeds. Varieties: **CD-R** (write once), **CD-RW** (rewritable), **CD-ROM** (read-only, written during production). |
| **DVD** (Digital Versatile/Video Disc) | Similar optical technology to CDs but with **higher storage capacity** (typically **4.7 GB to 9 GB**). Suited to storing digital videos which require more space than audio files or photos. |
| **Blu-Ray** | An advancement over DVDs with more than five times as much storage (typically **25 GB to 50 GB**), making them useful for storing high-resolution films. |

### Magnetic Storage

**Magnetic storage** devices represent binary information using two **magnetic states**: **polarised** and **unpolarised**. If a portion of magnetic material is polarised, all magnetic poles align and can be read by a **read/write head** passing over the region. If an area is unpolarised, magnetic poles are randomly scattered, producing a different reading. These two states represent **1 and 0**, allowing information to be stored in binary form.

| Device | Description |
|---|---|
| **Hard Disk Drive (HDD)** | The most common magnetic storage device. Typically has **high capacities** of between 500 GB and 5 TB (consumer-grade: 500 GB to 2 TB). Works by rotating **magnetic platters** at high speed under a read/write head on an **actuating arm**. Most HDDs have multiple platters stacked to maximise capacity. Disadvantages include somewhat **slow data transfer speeds** and susceptibility to **damage from movement** due to brittle platters and moving parts. |
| **Magnetic Tape** | First used to record computer data in the 1950s, popular through to the 1980s. Wound onto reels, long stretches of tape were passed through readers that checked the polarity and read off binary values. A **bulky** storage method. Improvements included reducing tape width and writing in diagonal lines instead of horizontal tracks, but it was eventually superseded by more modern technology. |
| **Floppy Disk** | A thin magnetic disk (originally flexible, hence the name) enclosed in plastic for protection from dust and dirt. Very portable due to thin size and low weight. Popular for exchanging small amounts of data, similar to how USB flash drives are used today. Typical storage capacity of **1 MB**, with some later versions reaching up to 200 MB. |

### Flash Storage

**Flash storage** uses **silicon semiconductors** forming the logic gates **NAND** and **NOR** to store electrical charge in one of two states: **high** or **low**. These two states represent binary values True (1) and False (0). Information is organised into **blocks**, which are combined to form **pages**.

**NOR** flash is preferred for storing small quantities of data such as code to be executed. **NAND** flash is preferred for larger files like photos and videos.

Flash memory can be **erased and reprogrammed electronically** and is **non-volatile** (retains data when power is lost). Although flash memory is similar to RAM in speed, they differ because RAM is **volatile** (loses content on power loss) while flash memory retains its information.

Flash memory is generally **more expensive per gigabyte** than magnetic and optical storage.

| Device | Description |
|---|---|
| **Solid State Drive (SSD)** | One of the most popular forms of data storage. Extremely **light and portable** with **no moving parts**, making them much more resistant to damage from movement than HDDs. Renowned for **high data transfer rates** and **silent operation**. Primary disadvantage is **cost** (though decreasing). Also has a **limited lifespan**: every time a page is written to, the voltage required to write to it increases, and over time this voltage becomes too high for the page to be written to. Typical consumer capacity: 120 GB to 4 TB. |
| **USB Flash Drive** | Portable flash storage typically ranging from 8 GB to 256 GB. Very high portability (small and lightweight). Moderate cost per gigabyte. Moderate durability (withstands casual handling but can be lost or damaged). |

### Storage Device Comparison

| Feature | HDD | SSD | USB Flash Drive | CD/DVD/Blu-Ray |
|---|---|---|---|---|
| **Type** | Magnetic | Flash | Flash | Optical |
| **Typical Capacity** | 500 GB - 2 TB | 120 GB - 4 TB | 8 GB - 256 GB | 700 MB / 4.7-9 GB / 25-50 GB |
| **Affordability** | Low cost per GB | High cost per GB | Moderate cost per GB | Low cost per disc |
| **Portability** | Low (especially internal) | High (especially external) | Very high | High (thin, lightweight) |
| **Durability** | Moderate (moving parts, shock risk) | High (no moving parts) | Moderate | Low (easily scratched) |
| **Speed** | Slower read/write | Fast read/write | Moderate | Slowest read/write |

### Choosing the Right Storage Device

**Storage needs**: how much data must be stored. Large volumes suit a high-capacity HDD; a few documents suit a USB flash drive.

**Performance needs**: whether fast data access is required. An SSD is best for tasks requiring high-speed access such as video editing or gaming.

**Portability**: whether data needs to be transported. USB flash drives and external SSDs are portable and suited to transferring data between computers.

**Cost**: higher-capacity and faster devices cost more. The budget must be weighed against storage and performance requirements.

## RAM and ROM

RAM and ROM are two types of **primary storage**, used by the computer's operating system to run the computer. They store information such as code instructions to execute and files required by running programs. This distinguishes them from **secondary storage devices** which store user data like images and videos.

### RAM (Random Access Memory)

**RAM** is a type of fast main memory used to store data and programs that the computer is **currently using**, such as open applications and files. The processor can both **read from and write to** RAM, allowing for fast data access and manipulation.

RAM is **volatile**, meaning it loses its contents when the computer is turned off or restarted. For this reason, RAM is used as **temporary storage** while the computer is running, not for storing personal files.

RAM is **directly accessible**, meaning the processor can access any part of the RAM equally fast (hence "Random Access"). RAM has **higher access speeds** than even flash memory but is **more expensive per gigabyte** than secondary storage, so computers typically have only 4 or 8 GB of RAM compared to hundreds or thousands of GB of secondary storage.

The more RAM a system has, the more data it can process simultaneously, leading to faster performance.

### ROM (Read-Only Memory)

**ROM** cannot be modified. Once programmed, the state of the memory cells inside does not change. Data stored in ROM can be read by the computer but **not written to**.

ROM stores the **BIOS** (Basic Input/Output System) or **firmware** used to start up the computer and initialise hardware components. This includes fixed sequences of instructions like the **bootstrap routine**. In mobile devices, ROM may contain the **bootloader** that starts the device and loads the operating system.

ROM is **non-volatile**, so it retains its data even when the computer is powered off. This is essential because the startup instructions must be preserved between power cycles.

### Comparing RAM and ROM

| Feature | RAM | ROM |
|---|---|---|
| **Purpose** | Stores data currently being used (open applications, files) | Stores BIOS/firmware for startup and hardware initialisation |
| **Volatility** | Volatile (loses contents on power loss) | Non-volatile (retains contents on power loss) |
| **Read/Write** | Can be read from and written to | Read-only (cannot be modified after programming) |
| **Speed** | Very fast access | Slower than RAM |
| **Typical Size** | 4 - 8 GB in consumer systems | Much smaller (stores only essential startup code) |

## Virtual Storage

**Virtual storage** (also known as **cloud storage**) is the name given to storing information **remotely** so that it can be accessed by any computer with access to the same system, for example over the Internet. Examples include cloud storage services like Google Drive and Microsoft OneDrive, as well as networked storage used in offices and schools. Data is typically hosted by **third-party providers** such as Microsoft, Google, or Apple.

### How Virtual Storage Works

Data is transmitted over the Internet to a **data centre**. The data centre uses **servers** to store data, which can be accessed and retrieved as needed. These servers are maintained by the cloud storage provider, who ensures the data is secure, backed up, and available for access.

Virtual storage is often an **abstraction of multiple drives** acting as one. Information stored in the cloud is actually stored on hundreds of hard drives or SSDs formatted to act as a single piece of storage.

### Benefits and Drawbacks

| Benefits | Drawbacks |
|---|---|
| **Scalability**: easy to increase storage capacity as needs grow | **Dependence on Internet connection**: access can be slow or impossible with a poor connection |
| **Accessibility**: data can be accessed from anywhere with an Internet connection, on any device | **Security concerns**: storing sensitive data off-site can pose a risk despite provider security measures |
| **Data protection**: data redundancy protects against data loss due to hardware failure | **Ongoing costs**: while cost-effective per unit, virtual storage is a recurring expense that accumulates over time |
| **Cost-effective**: pay only for the storage used, often more affordable than maintaining own infrastructure | **Network speed limitations**: transfer speeds are constrained by the user's Internet bandwidth |

### Virtual Storage vs Virtual Memory

These terms are easily confused in exams. **Virtual storage** refers to data stored remotely on the cloud. **Virtual memory** is a technique where secondary storage (e.g. part of an HDD or SSD) is used as an extension of RAM when the computer's physical RAM is fully utilised. They are entirely different concepts.

*(Beyond source: Virtual memory works by swapping pages of data between RAM and a designated area of secondary storage called a page file or swap file, managed by the operating system.)*
