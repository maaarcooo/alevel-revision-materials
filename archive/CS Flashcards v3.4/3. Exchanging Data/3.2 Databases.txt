What is a database? | An organised collection of data that allows easy storage, retrieval, and management of information
An organised collection of data that allows easy storage, retrieval, and management of information — what is this? | A database
What is an entity in a database? | An item of interest about which information is stored (e.g. students, orders, products)
An item of interest about which data is stored — what database term describes this? | An entity
What is an attribute in a database? | A characteristic of an entity; a category about which data is collected (i.e. a column/field)
A characteristic of an entity about which data is collected — what is this called? | An attribute (or field)
What is a field in a database? | A single piece of data in a record
What is a record in a database? | A group of related fields representing one complete data entry (i.e. one row in a table)
What is a table in a database? | A collection of records with a similar structure
What is a relational database? | A database that organises data into multiple linked tables, with each table representing a different entity
A database that organises data into multiple tables linked by keys — what type of database is this? | A relational database
What is a flat file database? | A database that stores all data in a single table, typically based around a single entity and its attributes
A database that consists of a single table — what is this called? | A flat file database
How is a flat file typically written out in notation? | Entity(Attribute1, Attribute2, Attribute3, ...) with the primary key underlined
How does a flat file differ from a relational database? | A flat file stores all data in one table (simple but causes redundancy and inconsistency). A relational database uses multiple linked tables (reduces redundancy and is easier to maintain)
Explain why a flat file database leads to data redundancy | In a flat file, data about related entities is stored in a single table, so information about one entity (e.g. a tutor's name) must be repeated across every record that references it, wasting storage space
Explain the problem with updating data in a flat file database | If a value changes (e.g. a tutor's name), every record containing that value must be found and updated. Missing any creates inconsistent data
What is a primary key? | A unique identifier for each record in a table, ensuring no two rows have the same value in that field
A unique identifier for each record in a table — what database term is this? | A primary key
How is a primary key shown in database notation? | By underlining it
What is a compound (composite) primary key? | A combination of two or more fields that together uniquely identify each record in a table
A primary key made up of two or more fields — what is this called? | A compound (composite) primary key
What is a foreign key? | An attribute in one table that refers to the primary key in another table, used to link tables and create relationships
An attribute that links two tables by referencing the primary key of another table — what is this called? | A foreign key
How is a foreign key shown in database notation? | Using an asterisk (*)
What is a secondary key? | A field (or fields) that is indexed to allow faster searching on that attribute
A field that is indexed to speed up database searches — what is this called? | A secondary key
Explain why secondary keys are useful in a database | Users rarely remember primary keys (e.g. PatientID). A secondary key indexes a more memorable field like surname, so the DBMS can search and order records on that field quickly without scanning every record
What is indexing in a database? | A method of storing the position of each record ordered by a certain attribute, used to speed up data retrieval
Explain how indexing speeds up data retrieval | Without an index, the DBMS must check every record sequentially. With an index, the DBMS can go directly to the relevant records, like looking up a topic in a book index instead of reading every page
The primary key is automatically [...] in a database | indexed
What is a Database Management System (DBMS)? | Software used to create, manage, and interact with databases (e.g. MySQL, Oracle, PostgreSQL, Microsoft SQL Server)
Software used to manage databases such as MySQL or PostgreSQL — what is this called? | A Database Management System (DBMS)
What are the three types of entity relationships? | One-to-one, one-to-many, and many-to-many
What is a one-to-one relationship? | A relationship where each entity in one table is linked to exactly one entity in another table (e.g. husband and wife)
What is a one-to-many relationship? | A relationship where one entity in a table can be linked to many entities in another table (e.g. a mother and her children)
What is a many-to-many relationship? | A relationship where entities in both tables can be linked to many entities in the other (e.g. students and courses)
How is a one-to-one relationship shown in an ER diagram? | A single straight line connecting the two entities
How is a one-to-many relationship shown in an ER diagram? | A line with a crow's foot (fork/branch) on the "many" side
How is a many-to-many relationship shown in an ER diagram? | A line with crow's feet (fork/branches) on both sides
What is an entity relationship diagram (ERD)? | A diagram that represents the entities (tables) in a database and the relationships between them
Why can many-to-many relationships not be directly implemented in a database? | They cannot be represented using simple foreign keys, so they must be resolved by creating a link table between the two entities
How is a many-to-many relationship resolved in a relational database? | By creating a link table (junction table) that sits between the two entities, converting the many-to-many into two one-to-many relationships
How can you tell from an ER diagram which table will contain a foreign key? | The entity on the "many" side of a relationship will have a foreign key linking to the primary key of the entity on the "one" side
What is normalisation? | The process of organising a relational database to reduce data duplication and improve data accuracy, consistency, and maintainability
The process of organising a database to eliminate redundancy and improve consistency — what is this called? | Normalisation
What does normalisation try to accomplish? | No redundancy (unnecessary duplicates), consistent data across linked tables, records can be added/removed without issues, and complex queries can be carried out
What are the requirements for First Normal Form (1NF)? | All values must be atomic (single, indivisible values in each cell), no repeating groups, unique column names, and a primary key must exist
What does "atomic" mean in the context of 1NF? | Each field must contain only a single, indivisible value — not lists or multiple values
A table has no repeating groups, all values are atomic, columns have unique names, and a primary key exists. What normal form is this in? | First Normal Form (1NF)
What are the requirements for Second Normal Form (2NF)? | The table must be in 1NF and have no partial dependencies — all non-key attributes must be fully dependent on the entire primary key
What is a partial dependency? | When a non-key attribute depends on only part of a composite primary key rather than the whole key
A table is in 1NF and all non-key attributes depend on the whole primary key (no partial dependencies). What normal form is this? | Second Normal Form (2NF)
When does 2NF only apply? | Only to tables that have a compound (composite) primary key. Tables with a single-field primary key that are in 1NF are automatically in 2NF
What are the requirements for Third Normal Form (3NF)? | The table must be in 2NF and have no transitive dependencies — every non-key attribute must depend only on the primary key and nothing else
What is a transitive dependency? | When a non-key attribute depends on another non-key attribute rather than directly on the primary key
A table is in 2NF and no non-key attribute depends on another non-key attribute (no transitive dependencies). What normal form is this? | Third Normal Form (3NF)
How does a partial dependency differ from a transitive dependency? | A partial dependency is when a non-key attribute depends on part of a composite primary key. A transitive dependency is when a non-key attribute depends on another non-key attribute
What is MICR (Magnetic Ink Character Recognition)? | A data capture method where characters printed in special magnetic ink are scanned and recognised by a computer, used on cheques
Characters printed in special magnetic ink that can be read by a computer — what technology is this? | Magnetic Ink Character Recognition (MICR)
What is OMR (Optical Mark Recognition)? | A data capture method that detects marked areas on paper, used for multiple choice tests, surveys, and lottery tickets
A technology that detects marked areas on specially designed forms — what is this? | Optical Mark Recognition (OMR)
What is OCR (Optical Character Recognition)? | A data capture method that converts printed or handwritten text into digital format
A technology that converts printed or handwritten text into digital text — what is this? | Optical Character Recognition (OCR)
How does OCR differ from OMR? | OCR recognises individual characters (letters/numbers) from text. OMR only detects the presence or position of marks in predefined areas on a form
What is data mining? | The process of discovering hidden patterns, correlations, and insights from large datasets using techniques from machine learning, statistics, and AI
The process of discovering hidden patterns and insights from large datasets — what is this called? | Data mining
What is selecting data in the context of data preprocessing? | Choosing only data that fits certain criteria to reduce the volume of input, and removing irrelevant information before storing it
What is Query By Example (QBE)? | A user-friendly visual interface method for constructing database queries, where users enter criteria in a grid representing database fields without writing SQL
A visual grid-based method of querying a database without writing SQL — what is this called? | Query By Example (QBE)
How does QBE differ from SQL for selecting data? | QBE uses a visual grid interface and is easier for non-technical users. SQL uses text-based commands and is more powerful and flexible for complex queries
What is EDI (Electronic Data Interchange)? | A standardised electronic method for transferring documents and data between businesses without human interaction
A standardised method for electronically exchanging business documents between computer systems — what is this? | Electronic Data Interchange (EDI)
What is CSV (Comma Separated Values)? | A simple data exchange format that stores tabular data in plain text, using commas to separate values
A plain text format that stores tabular data using commas as delimiters — what format is this? | CSV (Comma Separated Values)
What is JSON (JavaScript Object Notation)? | A lightweight, human-readable data interchange format that uses key-value pairs, commonly used in web applications
A lightweight data interchange format using key-value pairs, common in web applications — what is this? | JSON (JavaScript Object Notation)
What is an API (Application Programming Interface)? | A set of protocols that allow different software applications to communicate with each other and exchange data
A set of protocols enabling different software applications to communicate and share data — what is this called? | An API (Application Programming Interface)
What does SQL stand for? | Structured Query Language
What type of language is SQL? | A declarative language used to manipulate and interact with relational databases through a DBMS
A declarative programming language used to create, query, and manipulate databases — what language is this? | SQL (Structured Query Language)
What does the SELECT statement do in SQL? | Retrieves (collects) specified fields from a database table
What does the FROM clause do in SQL? | Specifies which table(s) the data should be retrieved from
What does the WHERE clause do in SQL? | Filters results based on a specified condition
What does ORDER BY do in SQL? | Sorts the results in ascending (default) or descending order based on a specified field
How do you sort results in descending order in SQL? | Add DESC after the ORDER BY field, e.g. ORDER BY DatePublished DESC
In SQL, values sorted with ORDER BY are in [...] order by default | ascending
What does the wildcard * mean in a SELECT statement? | Select all fields/columns from the table
What does the % wildcard do in a SQL LIKE clause? | Represents any sequence of characters in a pattern match (e.g. LIKE 'J%' matches any value starting with J)
What does the LIKE operator do in SQL? | Filters data based on a specific pattern, typically using % as a wildcard
What do AND and OR do in a SQL WHERE clause? | AND requires all conditions to be true. OR requires at least one condition to be true
What does the JOIN (INNER JOIN) command do in SQL? | Combines rows from two or more tables based on a related (common) field between them
What keyword follows JOIN to specify the linking condition? | ON, followed by the matching condition (e.g. ON Table1.Key = Table2.Key)
What does a nested SELECT do in SQL? | Places one SELECT statement inside another, using the result of the inner query as a condition for the outer query
Write the SQL syntax to select all fields from a table called Students where Age > 18 | SELECT * FROM Students WHERE Age > 18;
What does CREATE TABLE do in SQL? | Creates a new table in the database with specified attributes, data types, and constraints
What three things must be specified for each attribute when using CREATE TABLE? | Whether it is the primary key, its data type, and whether it must be filled in (NOT NULL)
What does NOT NULL mean in SQL? | The field must contain a value and cannot be left empty
What is CHAR(n) in SQL? | A string data type with a fixed length of exactly n characters
What is VARCHAR(n) in SQL? | A string data type with a variable length, up to a maximum of n characters
How does CHAR(n) differ from VARCHAR(n) in SQL? | CHAR(n) always stores exactly n characters (padding with spaces if needed). VARCHAR(n) stores only the actual characters entered, up to a maximum of n
What SQL data type stores TRUE or FALSE values? | BOOLEAN
What SQL data type stores whole numbers? | INTEGER (or INT)
What SQL data type stores numbers with a decimal point? | FLOAT
What SQL data type stores monetary amounts? | CURRENCY
What does ALTER TABLE do in SQL? | Adds, deletes, or modifies columns in an existing table
Write the SQL syntax to add a column Email of type VARCHAR(50) to a table called Users | ALTER TABLE Users ADD Email VARCHAR(50)
Write the SQL syntax to remove a column called Phone from a table called Users | ALTER TABLE Users DROP COLUMN Phone
What does INSERT INTO do in SQL? | Inserts a new record into a database table
Write the SQL syntax to insert a record with Name 'Ali' and Age 20 into a table called Students | INSERT INTO Students (Name, Age) VALUES ('Ali', 20)
What does UPDATE do in SQL? | Modifies existing records in a database table
Write the general SQL syntax for updating a record | UPDATE TableName SET column1 = value1 WHERE condition
What does DELETE FROM do in SQL? | Removes specific records from a database table based on a condition
Write the SQL syntax to delete all records from Products where Price < 5 | DELETE FROM Products WHERE Price < 5;
What does DROP TABLE do in SQL? | Deletes an entire table and all its data from the database
How does DELETE FROM differ from DROP TABLE? | DELETE FROM removes specific records from a table (the table still exists). DROP TABLE removes the entire table structure and all its data permanently
What is referential integrity? | The process of ensuring consistency between linked tables so that every foreign key value matches an existing primary key in the related table
Ensuring every foreign key matches a valid primary key in a linked table — what is this called? | Referential integrity
Explain how referential integrity can be broken | If a record containing a primary key is deleted while foreign keys in other tables still reference it, those foreign keys become orphaned (pointing to non-existent records), breaking consistency
What is a cascade action in referential integrity? | An automatic action applied to related records when a primary key is updated or deleted (e.g. CASCADE automatically deletes or updates linked foreign key records)
Name four types of cascade action for referential integrity | CASCADE (auto-updates/deletes related records), SET NULL (sets foreign key to null), SET DEFAULT (sets foreign key to default value), NO ACTION/RESTRICT (prevents the change if related records exist)
What is a transaction in database processing? | A sequence of database operations treated as a single unit of work — either all operations complete successfully or none do
A sequence of database operations treated as a single indivisible unit — what is this called? | A transaction
What does ACID stand for? | Atomicity, Consistency, Isolation, Durability
What is Atomicity in ACID? | A transaction must be processed in its entirety or not at all — if any part fails, the whole transaction is rolled back
A transaction must fully complete or not happen at all — which ACID property is this? | Atomicity
What is Consistency in ACID? | A transaction must maintain the database's referential integrity rules, moving it from one valid state to another
A transaction must maintain all database rules and constraints, keeping data valid — which ACID property is this? | Consistency
What is Isolation in ACID? | Simultaneous transactions must produce the same result as if they were executed one after another — intermediate states are not visible to other transactions
Concurrent transactions produce the same result as sequential execution — which ACID property is this? | Isolation
What is Durability in ACID? | Once a transaction has been committed, it will persist regardless of system failures (e.g. power cuts)
Committed transactions are permanently saved even after a system crash — which ACID property is this? | Durability
What is record locking? | A technique that prevents simultaneous access to the same database record, ensuring only one user can edit a record at a time to prevent inconsistencies or lost updates
Preventing simultaneous write access to the same record to avoid data inconsistencies — what technique is this? | Record locking
What is a shared lock (read lock)? | A lock that allows multiple transactions to read a record simultaneously but prevents modifications until released
What is an exclusive lock (write lock)? | A lock that allows only one transaction to access and modify a record, blocking all other reads and writes until released
How does a shared lock differ from an exclusive lock? | A shared lock allows multiple users to read simultaneously but blocks writes. An exclusive lock blocks all other access (both reads and writes) while one transaction modifies the record
What is deadlock in database processing? | A situation where two or more transactions are each waiting for the other to release a locked record, so neither can proceed
Two transactions each hold a lock the other needs, causing both to wait indefinitely — what is this called? | Deadlock
Explain how deadlock occurs with an example | User 1 locks Record A and needs Record B. User 2 locks Record B and needs Record A. Both wait for the other to release their lock, so neither can make progress
Name two techniques for resolving or preventing deadlock | Ordering resource requests (always lock records in a fixed order) and using a timeout mechanism (automatically roll back a transaction if it waits too long)
What is redundancy in the context of databases? | Storing one or more copies of the data in physically different locations so that if one copy is damaged, the data can be recovered from another
Storing copies of data in separate physical locations for recovery purposes — what is this called? | Redundancy
How does data redundancy in normalisation differ from redundancy for backup purposes? | In normalisation, redundancy means unnecessary duplicate data within a database (undesirable — wastes space and causes inconsistency). In backup/transaction processing, redundancy means deliberate copies in separate locations (desirable — protects against data loss)
What is a commit operation in transaction processing? | An operation that saves all changes made during a transaction as permanent
What is a rollback operation in transaction processing? | An operation that reverts all changes made during a transaction, restoring the database to its previous state
What is lock granularity? | The size of the data being locked, which can range from a single row to an entire table
What is timestamping in concurrency control? | Assigning a unique timestamp to each transaction to manage the order in which concurrent transactions are executed
What is the BETWEEN keyword used for in SQL? | Filtering results within a specified range (e.g. WHERE DatePublished BETWEEN #01/01/2000# AND #31/12/2005#)
Name three benefits of electronic databases over paper records | Easier to add, delete, modify, and update data. Data can be backed up and copied more easily. Multiple users from multiple locations can access the same database simultaneously
Name three advantages of barcodes for data capture | Fast and accurate data entry, reduces human error, and can be read quickly by machine
What are the two main types of barcodes? | 1D barcodes (parallel lines of varying widths, e.g. on products) and 2D barcodes (geometric patterns like QR codes that store more data)
