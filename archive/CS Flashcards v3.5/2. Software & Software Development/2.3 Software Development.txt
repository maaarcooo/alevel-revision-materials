What is the waterfall model? | A sequential software development process divided into distinct phases, where each phase must be completed before the next one begins
A sequential software development methodology where each phase must be completed before the next begins — what is this called? | The waterfall model (waterfall lifecycle)
What are the six stages of the waterfall lifecycle in order? | Requirements/Analysis, Design, Implementation, Testing, Deployment, Maintenance
What happens during the requirements/analysis stage of the waterfall model? | All possible system requirements are captured and documented clearly, and stakeholders state what they need from the finished product
What happens during the system design stage of the waterfall model? | Requirements are translated into a design — architects define the overall architecture and identify the main components
What happens during the implementation stage of the waterfall model? | The actual code is written based on the design documents, turning the system design into a functional program
What happens during the integration and testing stage of the waterfall model? | All components and modules are integrated and tested to ensure the entire system works as expected
What happens during the deployment stage of the waterfall model? | The product is released to the market or handed to the client, which may involve installation, customisation, and training
What happens during the maintenance stage of the waterfall model? | The system receives regular updates to fix bugs, improve performance, or add new features based on end-user feedback
What does TELOS stand for in the context of a feasibility study? | Technical, Economic, Legal, Operational, Scheduling
What does the Technical criterion in TELOS assess? | Whether the project is possible given the technology available and accessible
What does the Economic criterion in TELOS assess? | Whether the project can be financed in both the short-term and the long-term
What does the Legal criterion in TELOS assess? | Whether the project can be completed within the law
What does the Operational criterion in TELOS assess? | Whether the project can be successfully implemented and maintained
What does the Scheduling criterion in TELOS assess? | Whether the project can be completed given the time available
A feasibility study using the criteria Technical, Economic, Legal, Operational, and Scheduling — what acronym describes this? | TELOS
Give one benefit of the waterfall model | Simple and linear — easy to understand and follow, with each stage progressing sequentially to the next
Why does the waterfall model have clear milestones? | Each phase has specific deliverables that must be completed before moving on, making progress easy to measure
Give one drawback of the waterfall model related to flexibility | It is inflexible — changes are difficult to implement once development has started because the model does not easily allow revisiting previous stages
Explain why late-discovered problems are expensive to fix in the waterfall model | To make a change, programmers must revisit all stages between the current stage and the stage where the change is needed, which is costly and time-consuming
Why is there limited user involvement in the waterfall model? | Users are only involved at the very beginning (analysis) and end (evaluation) of the lifecycle, so they have little opportunity to provide feedback during development
What type of project is the waterfall model most suitable for? | Projects where requirements are well understood, unlikely to change, and where high quality and compliance are essential
What is an agile methodology? | A collection of software development methodologies that aim to improve flexibility and adapt to changes in user requirements faster through iterative development
A collection of methodologies that improve flexibility and adapt to changing user requirements through iterative development — what term describes this? | Agile methodology
What is extreme programming (XP)? | A type of agile software development methodology that promotes adaptability and high customer involvement, using pair programming and user stories
An agile model using pair programming alongside a representative end-user, built on user stories — what methodology is this? | Extreme programming (XP)
What are user stories in agile/XP development? | Requirements written from the user's perspective, typically in the format "As a user, I want to..." — used to capture functional and non-functional requirements
What is a sprint in agile development? | A short, time-boxed development period (usually 1-4 weeks) during which a set of tasks or features is completed
A short time-boxed development period of 1-4 weeks in agile — what is this called? | A sprint
What is pair programming? | A practice in XP where two programmers work together — one writes code while the other reviews and critiques it as it is written, aiming to produce high-quality code
What happens during sprint planning? | Requirements are broken down into tasks, a set of features is chosen for the current sprint, and the sprint goal is defined
What happens during a sprint review? | Working software is demonstrated to stakeholders, and feedback is collected to identify changes or improvements
What happens during a sprint retrospective? | The team reflects on what went well and what needs improving, focusing on team performance and process optimisation
How is testing handled in agile/XP compared to waterfall? | In agile/XP, testing is continuous throughout each sprint (unit, integration, and acceptance testing), whereas in waterfall, testing happens as a separate phase after implementation
What working-hour limit does XP recommend for programmers, and why? | No longer than 40 hours per week, with the aim that code quality is not compromised by fatigue
What is a "working version" in the context of XP? | Each iteration through the cycle produces a version of the program that could function as the final product
Give one benefit of agile/XP related to adaptability | It is highly adaptable — it can quickly respond to changes in requirements, even late in development
Give one benefit of agile/XP related to quality | It emphasises technical excellence and good design, with continuous testing throughout the development cycle
Give one drawback of agile/XP related to documentation | The focus on adaptability and immediate coding may result in insufficient or poor-quality documentation
What is scope creep in the context of agile development? | Uncontrolled changes or additions to requirements caused by the flexible nature of the methodology
Uncontrolled changes in requirements due to the flexible nature of an agile methodology — what term describes this? | Scope creep
Give one drawback of XP related to team structure | It requires experienced and skilled developers, and effective communication is essential — the end-user representative may not always be available
What type of project is agile/XP most suitable for? | Small to medium-sized projects where requirements can change and customer involvement is high
What is the spiral model? | A software development methodology that combines aspects of both iterative (agile) and sequential (waterfall) processes, with a focus on risk management
A software development model that combines iterative and sequential approaches with a focus on managing risk — what is this called? | The spiral model
What are the four key stages of the spiral model? | Planning (objectives/constraints), Risk Analysis (identify/mitigate risks), Engineering (develop/test/implement), Evaluation and Feedback (review with stakeholders)
What happens during the risk analysis stage of the spiral model? | Potential risks are identified and assessed, and mitigation strategies are planned
What happens if a project is found to be too risky at any point in the spiral model? | The project is terminated
How does the spiral model handle iterations? | The process repeats through its four stages in a spiral, with each loop representing a development phase, until the final product is ready
Give one benefit of the spiral model related to risk | It emphasises thorough risk assessment and mitigation, helping to identify and address issues early in development
Give one benefit of the spiral model related to customer involvement | It encourages feedback and input from clients throughout the development process and provides early partial working solutions
Give one drawback of the spiral model related to cost | It is expensive because it often requires hiring specialist risk assessors and involves constant prototyping
Why is the spiral model not suitable for small projects? | The extensive planning, risk management, and iterative processes involve overhead that is disproportionate for simple or small-scale projects
What type of project is the spiral model most suitable for? | Large, complex, risk-intensive projects with a high budget where requirements may change
What is Rapid Application Development (RAD)? | A software development methodology that emphasises fast and iterative development using prototypes that are continually built upon
A methodology that emphasises fast, iterative development using prototypes that are continually refined with user feedback — what is this called? | Rapid Application Development (RAD)
What are the stages of the RAD model? | Requirement Planning, User Design and Prototyping, Construction/Iterative Development, Cutover/Deployment, Maintenance and Updates
How are user requirements initially gathered in RAD? | Through focus groups and collaboration with users to develop prototypes
What happens during the construction stage of RAD? | The system is built incrementally with continuous user feedback and adaptation
What happens during the cutover stage of RAD? | The product is transitioned into the live environment, including user training, support, and documentation
Give one benefit of RAD related to speed | It enables rapid development and delivery of a high-quality system at a relatively low investment cost
Give one benefit of RAD related to user involvement | Clients are involved throughout development, ensuring the system aligns with user needs and expectations
Give one drawback of RAD related to code quality | The focus on speed may lead to skipping rigorous testing or producing inefficient code as requirements change over the course of the project
What type of project is RAD most suitable for? | Small to medium, low-budget projects requiring quick delivery where requirements can be developed and refined on the go, and where user requirements may be incomplete or unclear initially
How does the waterfall model differ from agile methodologies in handling changing requirements? | Waterfall is inflexible — changes require revisiting all stages back to where the change is needed. Agile welcomes changes at any point through iterative sprints and continuous feedback
How does the waterfall model differ from the spiral model in terms of risk management? | Waterfall has no dedicated risk analysis phase. The spiral model has risk analysis as one of its four core stages, with the project terminated if risks are too high
How does agile/XP differ from RAD? | Both are iterative, but XP emphasises pair programming, user stories, and continuous testing with a representative end-user. RAD emphasises rapid prototyping using focus groups and delivering partially functioning prototypes quickly
How does the spiral model differ from RAD? | Spiral focuses on managing risk-heavy, large-scale projects through formal risk analysis at every iteration. RAD focuses on speed and rapid prototyping for smaller projects with unclear requirements
Compare user involvement across waterfall and agile methodologies | In waterfall, users are involved only at the start (analysis) and end (evaluation). In agile, users are involved continuously through regular sprint reviews and feedback loops
Which development methodology produces the best documentation, and why? | Waterfall, because its sequential, phase-based structure naturally produces clear documentation at each stage. Agile and RAD tend to prioritise working software over documentation
What is alpha testing? | Testing carried out in-house by the software development teams within the company to find and fix bugs
Testing carried out in-house by the development team to identify and fix bugs — what type of testing is this? | Alpha testing
What is beta testing? | Testing carried out by end-users after alpha testing, where feedback is used to inform the next stage of development
Testing carried out by end-users after in-house testing, with feedback used to improve the software — what type of testing is this? | Beta testing
What is white box testing? | A form of testing where the test plan is based on the internal structure of the program, testing all possible routes through the code
Testing based on the internal structure of the program where all possible code paths are tested — what type of testing is this? | White box testing
What is black box testing? | A form of testing where the software is tested without the testers knowing the internal structure, tracing through inputs and outputs
Testing where the tester has no knowledge of the internal code structure and tests only inputs and outputs — what type of testing is this? | Black box testing
How does white box testing differ from black box testing? | White box testing is based on the internal structure of the program and tests all code paths. Black box testing is based on inputs and outputs only, with no knowledge of internal code
What happens during the evaluation stage of a software development lifecycle? | The effectiveness of the software is evaluated against the system requirements defined during analysis, considering criteria such as robustness, reliability, portability, and maintainability
What four criteria are commonly used to evaluate software after implementation? | Robustness, reliability, portability, and maintainability
In the waterfall model, at which stage is a feasibility study conducted? | During the analysis stage
What is a software development life cycle (SDLC)? | A structured process that defines the stages involved in developing software, from initial analysis through to maintenance
An algorithm is a [...] used to solve a problem | Set of instructions
Name three key qualities of a good algorithm | Inputs must be clearly defined, it must always produce a valid output for any defined input, and it must always reach a stopping condition
Why must an algorithm be well-documented and well-commented? | Documentation allows it to be used as a reference, and comments make it easier for others to understand and modify the code
In the context of agile, what is the difference between a sprint review and a sprint retrospective? | A sprint review demonstrates working software to stakeholders and collects feedback on the product. A sprint retrospective focuses internally on the team's process, reflecting on what went well and what to improve
Which methodologies are iterative: waterfall, agile, spiral, or RAD? | Agile, spiral, and RAD are all iterative. Waterfall is sequential (non-iterative)
In agile development, how are sections of a project developed? | The problem is broken down into sections which are developed in parallel, meaning different parts of the software can be at different stages of development simultaneously
Why might the spiral model lack focus on code efficiency? | Because the emphasis is on risk management and prototyping rather than optimising the codebase, the frequent iterations may introduce inefficient code
What is the purpose of a test plan in software development? | To define in advance what tests will be carried out, what inputs will be used, and what outputs are expected, so the software can be systematically verified against requirements
During which SDLC stage is a test plan typically designed? | During the design stage
What is meant by the term "prototype" in the context of RAD or agile? | A partially functioning version of the software that is built early and refined through user feedback until it meets requirements
