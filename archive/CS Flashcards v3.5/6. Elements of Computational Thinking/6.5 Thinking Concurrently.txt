What is concurrent processing? | Multiple processes/tasks running on a single processor, where each process is given a time slice of processor time before swapping to the next process
Multiple processes share a single processor by each receiving a fraction of processor time before control is swapped — what is this called? | Concurrent processing
What is parallel processing? | Multiple processes running at the same time on multiple processor cores to enable faster computation
Multiple processes executing simultaneously across multiple processor cores — what is this called? | Parallel processing
How does concurrent processing differ from parallel processing? | Concurrent processing runs multiple tasks on a single processor by interleaving time slices (tasks execute sequentially but appear simultaneous). Parallel processing runs multiple tasks genuinely at the same time on multiple processor cores
What is concurrent thinking? | The mindset of identifying patterns and parts of problems where concurrency can be applied — determining which tasks can be tackled at the same time
The ability to spot parts of a problem that can be dealt with at the same time — what term describes this? | Concurrent thinking
How does concurrent thinking differ from concurrent processing? | Concurrent thinking uses your brain to identify where concurrency applies. Concurrent processing uses a computer processor to interleave tasks via time slicing
What are dependencies in the context of concurrent processing? | Tasks that rely on other tasks in order to either start or complete
Tasks that must wait for other tasks to finish before they can begin or be completed — what term describes these? | Dependencies
Give an example of a dependency that prevents concurrent execution | A roof cannot be put on a house until the walls are fully built, and walls cannot be built until the foundation is in place
Why must developers consider dependencies when designing concurrent solutions? | Because tasks with dependencies cannot run at the same time — a dependent task must wait until the task it relies on is complete before it can start
What is a time slice in concurrent processing? | A small fraction of processor time allocated to a process before the processor swaps to another process
Why does concurrent processing appear simultaneous to a user even though it is sequential? | Each process is allocated only microseconds of execution time, and human brains process information too slowly to perceive the rapid switching between tasks
When can data sets in a batch program be run in parallel on multiple cores? | When the data sets do not relate to, rely on, or interact with each other (i.e. they have no dependencies)
What does pipelining mean in the context of concurrent processing? | Multiple batch programs are interchanged on the same processor, each receiving time slices, allowing all programs to make progress within a shorter timeframe
State one benefit of concurrent processing related to throughput | The number of tasks completed in a given timeframe is increased, as multiple programs can make partial progress rather than each waiting for the previous one to finish
State one benefit of concurrent processing related to idle time | Time spent waiting for user input or I/O is minimised, because the processor swaps to another task and waits for an OS interrupt before swapping back to the waiting task
State one drawback of concurrent processing related to completion time | If large numbers of users or processes involve heavy computation, individual processes take longer to complete because each is only allocated a limited time slice
State one drawback of concurrent processing related to overhead | There is overhead in coordinating and switching between processes (context switching), which reduces overall program throughput
Why are some tasks not suited to concurrent processing? | Not all tasks can be meaningfully broken up and interleaved — some tasks have strong sequential dependencies or perform worse when frequently interrupted
State one benefit of parallel processing related to repetitive calculations | Parallel processing can speed up performance on repetitive calculations over large data sets (e.g. image/video editing) by splitting the work across several processors
How do GPUs make use of parallel processing? | GPUs split the computations needed to render 3D scenes between multiple cores, allowing graphical calculations to be performed simultaneously
State one drawback of parallel processing related to communication | Different processors running simultaneously may need to communicate with each other, leading to overhead and delays in processing
State one drawback of parallel processing related to optimisation | Some tasks may run faster or may be optimised to run on a single processor rather than multiple processors, so parallel execution provides no benefit
How can scheduling algorithms affect concurrent processing? | Some OS scheduling algorithms can prioritise certain concurrent processes over others, meaning higher-priority or larger processes could finish sooner
When determining which parts of a problem can be solved concurrently, what should you assess first? | Which parts of the problem are related — related sections can often be solved simultaneously and dealt with concurrently
In a worked exam answer about concurrent processing, what three elements should you include? | State your example, explain how it acts concurrently (time-sliced on one processor), and explain why it is necessary that it runs concurrently
In a flight simulator, explain how weather simulation and plane movement could use concurrent processing | Weather conditions and plane movement are largely independent (no constant dependency), so they can be given alternating time slices on one processor, allowing both to update and appear to run simultaneously
What happens in a system without concurrent processing when a long-running task is executing? | No other process can run until that task finishes completely — e.g. a user would have to wait for a virus scan to finish before they could play a game
How does concurrent processing handle a process that is waiting for user input? | The processor swaps to another task and continues executing it, then an OS interrupt signals when the waiting task has received its input, and the processor swaps back
