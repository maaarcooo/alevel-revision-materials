What is branching? | A programming control structure where the code selects one or more alternative paths depending on the evaluation of a boolean expression
A control structure that selects alternative paths based on evaluating a boolean expression — what is this called? | Branching
What is a function in programming? | A subroutine that performs a task or calculation and always returns a value. It can be called in an expression or assigned to a variable
A subroutine that always returns a value and can be used in an expression or assigned to a variable — what is this called? | A function
What is a global variable? | A variable declared in the main program, outside any subroutines, but accessible anywhere in the program
A variable declared in the main program that exists outside subroutines but can be used anywhere — what term describes this? | Global variable
What is an Integrated Development Environment (IDE)? | A software package that helps users develop and implement code more easily, with features for editing, debugging, version control, testing, and compilation
A software package providing editing, debugging, version control, testing, and compilation features for code development — what is this called? | An Integrated Development Environment (IDE)
What is iteration in programming? | A control structure where a set of statements is repeated either a fixed number of times or until a condition is met
A control structure that repeats statements a fixed number of times or until a condition is met — what is this called? | Iteration
What is a local variable? | A variable declared within a subroutine that only exists and can be used within that subroutine
A variable that only exists within the subroutine where it is declared — what term describes this? | Local variable
What is modularity? | The technique of breaking down a complex problem into simpler, self-contained components called modules, each implementing a specific subtask required to solve the problem
Breaking a complex problem into simpler, self-contained modules that each implement a specific subtask — what technique is this? | Modularity
What is Object Oriented Programming (OOP)? | A programming paradigm where the system is viewed as a set of objects, each with their own attributes and methods, that can interact with each other. All processing is done by objects
A paradigm where the system is a set of interacting objects, each with attributes and methods, and all processing is done by objects — what is this called? | Object Oriented Programming (OOP)
What is parameter passing by reference? | Passing the address or pointer of the required value into a procedure, so the original variable can be modified
Passing the address or pointer of a value into a procedure rather than a copy — what type of parameter passing is this? | Parameter passing by reference
What is parameter passing by value? | Creating a temporary local copy of the actual value of a variable and passing it into the procedure
Creating a temporary local copy of a variable's value to pass into a procedure — what type of parameter passing is this? | Parameter passing by value
How does parameter passing by reference differ from passing by value? | By reference passes the address/pointer, so changes affect the original variable. By value passes a temporary copy, so the original variable is unaffected
What are parameters? | The data values required to be passed into a subroutine
What is a procedure? | A subroutine called by writing its name in the code that does not have to return a value
A subroutine that is called by name and does not have to return a value — what is this called? | A procedure
How does a function differ from a procedure? | A function always returns a value and can be used in expressions or assigned to variables. A procedure does not have to return a value and is called simply by writing its name
What is recursion? | A programming technique where a subroutine calls itself repeatedly until a base case is met. The base case must be chosen to avoid infinite loops
A subroutine that calls itself until a base case is reached — what is this called? | Recursion
Why must a recursive subroutine have a base case? | To ensure the recursion terminates, preventing an infinite loop
What is a sequence in programming? | A control structure in which statements are executed one after another in the order they appear in the script
A control structure where statements execute one after another in the order they appear — what is this called? | Sequence
What is a subroutine? | A uniquely named section of code written to perform a specific task within a program
A uniquely named section of code that performs a specific task within a program — what is this called? | A subroutine
What are the three main programming control structures? | Sequence, branching (selection), and iteration
What is backtracking? | An algorithm that incrementally finds a solution by trying different sequences and abandoning a path when it cannot lead to a valid solution
An algorithm that tries different paths incrementally and abandons any path that cannot lead to a valid solution — what is this called? | Backtracking
What is a computable problem? | A problem for which every instance can be solved in a finite number of steps by means of an algorithm
A problem where every instance is solvable in a finite number of steps using an algorithm — what term describes this? | A computable problem
What are computational methods? | Methods of solving problems using some form of computation by devising and implementing an algorithm
What is data mining? | An algorithm that finds solutions by analysing large data sets to uncover trends and relationships between variables
An algorithm that analyses large data sets to uncover trends and relationships between variables — what is this called? | Data mining
What is divide and conquer? | An algorithm design technique that decomposes problems by reducing the problem size with each iteration until the sub-problems become solvable
An algorithm design technique that repeatedly reduces problem size until sub-problems become solvable — what is this called? | Divide and conquer
What are heuristics? | 'Rule of thumb' algorithms that produce a valid but sub-optimal solution for hard (intractable) problems as an approximation
A 'rule of thumb' algorithm that produces a valid but non-optimal solution for an intractable problem — what is this called? | A heuristic
Why are heuristics used instead of exact algorithms for some problems? | Because the problem is intractable (cannot be solved optimally in a reasonable time), so a heuristic provides a good-enough approximate solution
What is performance modelling? | The process of simulating the behaviour of a model under different virtual user and system loads using mathematical approximation
Simulating a system's behaviour under different virtual user and system loads by mathematical approximation — what is this called? | Performance modelling
What is pipelining? | The process of splitting a task into parts and then searching for subtasks that can be processed simultaneously, overlapping the processing of each part
Splitting a task into parts and overlapping the processing of subtasks that can run simultaneously — what is this called? | Pipelining
What is problem decomposition? | The process of splitting a problem into smaller, solvable sub-problems that are easier to conceive, understand, maintain, and program
Splitting a problem into smaller, more manageable sub-problems — what is this called? | Problem decomposition
What is problem recognition? | The ability to recognise the most effective strategy to solve a problem
The ability to identify the most effective strategy for solving a problem — what is this called? | Problem recognition
What is visualisation in the context of computational methods? | Using a visual representation of an algorithm or data structure to translate a problem and its solution into a more human-readable form
Using visual representations to make algorithms or data structures more human-readable — what computational method is this? | Visualisation
How does backtracking differ from divide and conquer? | Backtracking tries different paths and abandons invalid ones to find a solution. Divide and conquer breaks a problem into smaller sub-problems, reducing the size with each iteration until they are directly solvable
How does problem decomposition relate to modularity? | Problem decomposition splits a problem into smaller sub-problems. Modularity implements those sub-problems as self-contained modules, each handling a specific subtask
